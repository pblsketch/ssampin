# 📌 쌤핀 (SsamPin) — Product Requirements Document

> "중요한 걸 핀으로 꽂아두는 선생님의 대시보드"

**버전:** v0.1 (초안)
**최종 수정:** 2026-02-27
**작성자:** 준일

---

## 1. 프로젝트 개요

### 1.1 비전

교사가 매일 반복적으로 확인하고 관리하는 정보들을 한 화면에 모아주는 데스크톱 앱.
바탕화면에 항상 띄워두는 "교사의 정보 허브" 역할.

### 1.2 한 줄 요약

바탕화면에 항상 띄워두는 교사용 대시보드. 시계, 시간표, 일정, 좌석배치, 메모를 한 화면에. 풀 앱 모드 + 컴팩트 위젯 모드 지원.

### 1.3 이름의 의미

- **쌤** = 선생님의 애칭
- **핀** = 중요한 것을 핀으로 꽂아두다, 대시보드에 고정(pin)하다
- 교사가 중요한 정보를 핀꽂듯 대시보드에 꽂아두고 한눈에 관리

## 2. 타겟 사용자

### 2.1 주 사용자 (Primary)

**중등학교 교사 (중학교/고등학교)**
- 여러 학급에 가서 수업하는 교과 교사 (국어, 영어, 수학 등)

- IT 전문가가 아닌 일반 교사 → 설치·설정이 간단해야 함

- 학교 일정과 나무학교 일정을 동시에 관리해야 하는 수요

## 3. 문제 정의

### 3.1 현재의 불편함

| # | 문제 | 현재 해결 방법 | 한계 |
|---|------|---------------|------|
| 1 | 시간표 확인 | PPT/종이 시간표 프로젝터에 띄움 | 수정할 때마다 파일 편집 필요, 현재 교시 표시 없음 |
| 2 | 오늘 일정 확인 | 종이 월중행사표, 교무실 칠판 | 교실에서 바로 확인 불가, 지난 일정 놓침 |
| 3 | 좌석 배치 관리 | HWP/Excel로 매번 새로 작성 | 자리 바꿀 때마다 문서 편집 번거로움 |
| 4 | 행사 일정 놓침 | 수기 메모, 기억에 의존 | 특히 D-3, D-1 사전 준비가 필요한 행사를 놓침 |
| 5 | 시계 확인 | 교실 벽시계 or 컴퓨터 우하단 | 프로젝터로 크게 보여줄 수 없음 |
| 6 | 메모 관리 | 포스트잇, 수첩 | 흩어져서 관리 안 됨 |
| 7 | 오늘의 메시지 | 칠판에 매번 작성 | 프로젝터 화면에 통합 불가 |

### 3.2 핵심 니즈

1. **한 화면에서 다 보이게** — 시계, 날씨, 시간표, 일정, 메시지를 한눈에
2. **현재 상태 자동 표시** — 지금 몇 교시인지, 오늘 날씨, D-Day 자동 계산
3. **시인성 최적화** — 어두운 배경, 큰 글씨, 위젯 모드에서도 한눈에 파악
4. **간단한 관리** — 시간표·좌석·일정 입력이 복잡하지 않은 UI
5. **오프라인 동작** — 학교 네트워크 불안정해도 핵심 기능 사용 가능

## 5. 기능별 상세 요구사항

### 5.1 메인 대시보드

바탕화면에 항상 띄워두는 "홈 화면". 두 가지 모드 지원:
- **풀 앱 모드**: 사이드바 포함 전체 대시보드 (업무 시)
- **컴팩트 위젯 모드**: 바탕화면 모서리에 핵심 정보만 표시 (상시 모니터링용)

#### 화면 구성 (위→아래)

```
┌──────────────────────────────────────────────┐
│          2월 27일 (금) 16:35                  │  ← 큰 날짜/시간
│    ☁️ 2°C~10°C  💧85%  😷 미세 4 초미세 4      │  ← 날씨/미세먼지
│                                              │
│  ┌─ 오늘의 일정이 모두 끝났습니다! 수고하셨습니다 👏 ─┐  │  ← 메시지 배너
│                                              │
│  ┌── 일주일 시간표 ──┐  ┌── 학교 교육활동 계획 ──┐  │  ← 좌: 시간표, 우: 일정
│  │  월 화 수 목 금   │  │ 3/1(일) 교내방역     │  │
│  │ 1 국어 과학 ...   │  │ 3/3(화) 시업식 ...   │  │
│  │ 2 영어 음악 ...   │  │ 3/4(수) 안전점검 ... │  │
│  │ ...              │  │ ...                │  │
│  │ 1교시 8:50~9:30  │  │                    │  │
│  └──────────────────┘  └────────────────────┘  │
└──────────────────────────────────────────────┘
```

#### 요구사항

- **FR-DASH-01**: 현재 날짜를 `M월 D일 (요일)` 형식으로 표시. 시간을 `HH:MM` 형식으로 초 단위 갱신.
- **FR-DASH-02**: 날씨 정보 표시 — 날씨 아이콘, 최저/최고기온, 습도, 미세먼지/초미세먼지 등급.
- **FR-DASH-03**: "오늘의 메시지" 배너. 클릭하면 인라인 편집 모드. Enter로 저장, Esc로 취소.
- **FR-DASH-04**: 일주일 시간표를 5열(월~금) × N행(교시) 테이블로 표시. 오늘 요일 열 강조. 현재 교시 셀 하이라이트(노란색/초록색 배경).
- **FR-DASH-05**: 학교 교육활동 계획을 날짜순 리스트로 표시. 이번 달~다음 달 범위. 일정 타입별 색상 구분.
- **FR-DASH-06**: D-Day 이벤트가 있으면 카운트다운 배지 표시 (예: "D-18 중간고사").
- **FR-DASH-07**: 교시 시간표(1교시 8:50~9:30 등)를 시간표 하단에 표시.

#### 컴팩트 위젯 모드 요구사항

- **FR-WIDGET-01**: 풀 앱 ↔ 위젯 모드 전환 버튼 (타이틀바 또는 트레이 메뉴).
- **FR-WIDGET-02**: 위젯 크기 자유 조절. 기본 350×500px. 사이드바 없음.
  - 모서리/변 드래그로 자유롭게 리사이즈 가능.
  - 최소 크기: 280×350px (핵심 정보 가독성 확보).
  - 최대 크기: 제한 없음 (풀 앱 수준까지 키울 수 있음).
  - 크기에 따라 콘텐츠 반응형 표시:
    - **소형** (~350px 폭): 시계+시간표+일정만 표시
    - **중형** (~450px 폭): + 할일 + 메모 미리보기 추가
    - **대형** (550px+ 폭): 모든 섹션 + 좌석 미니뷰까지 표시
  - 마지막 크기 기억 (재실행 시 유지).
- **FR-WIDGET-03**: 위젯 표시 항목: 날짜/시계, 날씨, 오늘 시간표(현재 교시 하이라이트), 다가오는 일정 2~3개.
- **FR-WIDGET-04**: **배경 투명도 조절** — 슬라이더(0~100%)로 위젯 배경 투명도 조절 가능. 기본값 80%.
  - Electron의 BrowserWindow transparent + opacity 조합.
  - 투명도 설정은 위젯 우클릭 컨텍스트 메뉴 또는 설정에서 조절.
- **FR-WIDGET-05**: Always on Top 기본 활성화 (위젯 모드에서).
- **FR-WIDGET-06**: 바탕화면 위치 기억 (마지막 위치에서 재실행).
- **FR-WIDGET-07**: 더블클릭 시 풀 앱 모드로 전환.

#### 디자인 요구

- 배경: 어두운 네이비/차콜 (#0a0e17 계열)
- 시계: 최소 60px 이상 대형 폰트, 흰색
- 날씨: 시계 바로 아래, 아이콘+텍스트 가로 배치
- 메시지 배너: 틸/에메랄드 계열 반투명 배경, 둥근 모서리
- 시간표: 과목별 배경색 구분 (국어=노랑, 영어=초록, 수학=파랑 등)
- 위젯 모드: 반투명 배경 + 둥근 모서리 + 블러 효과

### 5.2 시간표

#### 두 가지 모드

1. **학급 시간표**: 우리 반 이번 주 시간표. 단순히 교시별 과목명만 표시.
2. **교사 시간표**: 내가 어디서 몇 교시에 수업하는지. 과목명 + 학급명(예: "국어 / 2-3") 표시.

#### 요구사항

- **FR-TT-01**: 학급/교사 모드를 탭으로 전환.
- **FR-TT-02**: 5열(월~금) × N행(교시) 격자. 행 수는 설정의 교시 수에 따라 동적.
- **FR-TT-03**: 현재 요일의 열을 시각적으로 강조 (헤더 색상 변경).
- **FR-TT-04**: 현재 교시의 셀을 하이라이트. 색상: 노란색 계열 + 테두리/글로우 효과.
  - "현재 교시" 판단 기준: 설정의 교시 시간(시작~종료)과 현재 시각 비교.
  - 점심시간이면 하이라이트 없음.
  - 주말이면 하이라이트 없음.
- **FR-TT-05**: 과목별 배경 색상 매핑. 기본 매핑 제공 + 설정에서 커스텀 가능(v2.0).
  - 기본: 국어=노랑, 영어=초록, 수학=파랑, 과학=보라, 사회=주황, 체육=빨강, 음악=분홍, 미술=남색, 창체=청록
- **FR-TT-06**: 교사 시간표에서 "내 수업"(내 과목+학급 지정된 셀)은 강조, 비어있거나 다른 과목은 흐리게.
- **FR-TT-07**: 편집 모드 진입 시 각 셀이 input 필드로 변환. 학급 시간표는 과목명만, 교사 시간표는 과목명+학급명.
- **FR-TT-08**: 교시 시간 정보를 시간표 하단에 표시 (1교시 8:50~9:30, ..., 점심 12:00~12:50).

### 5.3 좌석 배치도

#### 요구사항

- **FR-SEAT-01**: N×M 격자(기본 6열×6행, 설정에서 변경 가능). "교탁" 라벨을 격자 상단에 표시.
- **FR-SEAT-02**: 각 좌석에 학번 + 이름 표시. 빈 좌석은 "빈 자리"로 점선 표시.
- **FR-SEAT-03**: 드래그앤드롭으로 두 좌석을 교환. 학생이 있는 좌석끼리, 또는 학생↔빈자리 교환 가능.
- **FR-SEAT-04**: 편집 모드에서 각 좌석의 학번·이름을 직접 입력/수정.
- **FR-SEAT-05**: 하단에 "총 N명 / M석" 통계 표시.
- **FR-SEAT-06**: 좌석 랜덤 배치 기능 (버튼 클릭 시 현재 학생들을 무작위 재배치). — v1.0
- **FR-SEAT-07**: 좌석 배치 히스토리(최근 5개). 이전 배치로 되돌리기. — v2.0

### 5.4 학교 일정 + 나무학교 일정

#### 요구사항

- **FR-EVT-01**: "학교 일정"과 "나무학교 일정"을 탭으로 분리.
- **FR-EVT-02**: 월별 필터 (1월~12월 버튼). 선택한 월의 일정만 리스트로 표시.
- **FR-EVT-03**: 각 일정 항목: 날짜, 요일, 타입 라벨(학교/공휴일/시험/방학/나무학교), 제목.
- **FR-EVT-04**: 일정 타입별 색상 라벨:
  - 학교: 파랑
  - 공휴일: 빨강
  - 시험: 노랑
  - 방학: 초록
  - 나무학교: 보라
- **FR-EVT-05**: D-Day 플래그가 있는 일정은 남은 날짜를 `D-N` 형식으로 표시.
- **FR-EVT-06**: 일정 추가 폼: 날짜(date picker), 제목(text), 타입(select), D-Day 여부(checkbox).
- **FR-EVT-07**: 일정 삭제: 호버 시 삭제 버튼 표시. 확인 없이 즉시 삭제 (Undo 스낵바로 복구 가능 — v1.0).
- **FR-EVT-08**: 대시보드에는 이번 달~다음 달 일정을 날짜순으로 표시.

### 5.5 행사 팝업 알림

#### 요구사항

- **FR-ALERT-01**: 앱 실행 시 오늘 날짜 기준으로 알림 대상 일정을 확인.
- **FR-ALERT-02**: 알림 조건: 일정의 날짜가 오늘(D-Day), D-1, D-3인 경우.
  - 알림 일수는 기본값 [0, 1, 3]. 향후 설정에서 커스텀 가능(v2.0).
- **FR-ALERT-03**: 알림이 있으면 모달 팝업으로 표시. 내용: D-Day 라벨 + 행사 제목 리스트.
- **FR-ALERT-04**: "확인" 버튼 클릭 시 팝업 닫힘. 같은 날 다시 실행해도 재표시하지 않음 (로컬 플래그).
- **FR-ALERT-05**: 하루가 지나면 플래그 리셋 (다음 날 실행 시 새 알림 표시).

### 5.6 일정 공유 (파일 기반)

한 명이 만든 일정을 다른 선생님들에게 파일로 공유하여 일괄 등록하는 기능.
예: 나무학교 담당자가 연간 일정을 만들어 `.ssampin` 파일로 공유 → 다른 선생님들이 한 번에 가져오기.

#### 내보내기 (공유하기)

- **FR-SHARE-01**: 일정 관리 페이지에서 "📤 일정 공유" 버튼.
- **FR-SHARE-02**: 공유 범위 선택:
  - 카테고리별 선택 (예: "나무학교" 일정만 공유, "학교" 일정만 공유)
  - 기간 선택 (전체 / 이번 학기 / 특정 월)
  - 개별 일정 체크박스 선택
- **FR-SHARE-03**: `.ssampin` 파일 생성 (JSON 기반, 확장자 `.ssampin`).
  - 파일 내용: 메타데이터(작성자, 생성일, 설명) + 카테고리 정보 + 일정 배열
  - 사람이 읽을 수 있는 JSON 형식
- **FR-SHARE-04**: 파일 저장 대화상자 → 원하는 위치에 저장.
- **FR-SHARE-05**: 파일명 자동 제안: `나무학교_2026_일정.ssampin`, `학교_3월_일정.ssampin` 등.

#### 가져오기 (받기)

- **FR-SHARE-06**: 일정 관리 페이지에서 "📥 일정 가져오기" 버튼.
- **FR-SHARE-07**: `.ssampin` 파일 선택 → 미리보기 팝업:
  - 파일 정보: 작성자, 생성일, 설명
  - 포함된 일정 개수 및 목록 미리보기
  - 카테고리 매핑: 파일의 카테고리 → 내 카테고리로 매핑 (자동 매칭 + 수동 변경 가능)
  - 중복 처리: 같은 날짜+제목 일정이 이미 있으면 표시 (덮어쓰기/건너뛰기 선택)
- **FR-SHARE-08**: "가져오기" 클릭 시 선택한 일정을 내 일정에 추가.
- **FR-SHARE-09**: 가져오기 완료 시 결과 요약: "N개 추가, M개 건너뜀, K개 업데이트".
- **FR-SHARE-10**: `.ssampin` 파일 더블클릭 시 쌤핀이 열리며 가져오기 팝업 자동 표시.
  - Electron의 파일 연결(file association) 기능으로 `.ssampin` 확장자 등록.
  - 앱이 이미 실행 중이면 기존 창에서 팝업.

#### 공유 시나리오

1. **나무학교 담당자**: "일정 공유" → 나무학교 카테고리 선택 → `나무학교_2026_연간일정.ssampin` 저장 → 카톡 그룹에 파일 공유
2. **다른 선생님들**: 파일 다운로드 → 더블클릭 → 쌤핀 열림 → 미리보기 확인 → "가져오기" → 완료!
3. **학교 교무부**: 월중행사표를 `.ssampin`으로 만들어 교직원 단체방에 공유 → 전 교사가 한 번에 등록

### 5.7 담임 메모장 (학생 관리)

담임 교사가 학생별 출결, 상담, 생활 기록을 관리하는 기능.
레퍼런스: 학생관리 앱 (script.google.com 기반 학생관리 도구)

#### 화면 구성

**상단:**
- 수업 반 선택 탭: [담임] [1-1] [1-2] [1-3] ... (담당 학급 선택)
- 탭 하위: [입력] [진도] [조회] 3가지 모드
- "담임" 탭에서는 담임반 학생 전체, 다른 탭은 해당 수업반 학생

**입력 모드:**

학생 선택 영역:
- 학급 학생 전체를 번호순 격자로 표시 (5열 × N행)
- 각 학생: 학번 + 이름 버튼
- 선택된 학생: 파란 하이라이트
- 다중 선택 가능 (같은 사유로 여러 학생 기록 시)

카테고리 선택 (태그 칩 방식):
- **출결**: 생리결석, 병결, 무단결석, 지각, 조퇴, 결과
- **상담/관계**: 학부모상담, 학생상담, 교우관계
- **생활/학습**: 보건, 생활지도, 학습, 칭찬
- **기타**: 진로, 가정연락, 기타
- 카테고리만 선택해도 바로 저장 가능

메모 입력 (선택):
- 텍스트 영역: "추가 메모가 있으면 입력하세요..."
- 선택사항 — 카테고리만으로도 간단 기록 가능

저장:
- "저장하기" 버튼 → 선택한 학생 × 카테고리 × 메모로 기록 생성
- 저장 시 자동으로 날짜/시간 기록

**조회 모드:**

- 학생별 필터: 특정 학생 선택 시 해당 학생의 전체 기록 타임라인
- 카테고리별 필터: 출결만, 상담만 등
- 기간별 필터: 이번 주 / 이번 달 / 전체
- 각 기록: 날짜 + 카테고리 태그 + 메모 + 편집/삭제 버튼
- 통계 요약: 학생별 출결 현황 (결석 N회, 지각 N회 등)

**상벌점 모드 (v2.0):**
- 상점/벌점 부여 및 누계 관리
- 학교별 상벌점 기준 커스텀 설정

#### 요구사항

- **FR-STMEMO-01**: 좌석배치의 학생 데이터와 연동. 좌석배치에 등록된 학생 목록을 자동으로 가져옴.
- **FR-STMEMO-02**: 학생 선택 → 카테고리 선택 → (선택적 메모) → 저장의 3단계 흐름.
- **FR-STMEMO-03**: 카테고리는 기본 제공 + 사용자 정의 추가 가능.
- **FR-STMEMO-04**: 다중 학생 동시 선택하여 일괄 기록 가능 (예: 3명 동시 지각 처리).
- **FR-STMEMO-05**: 기록 날짜는 자동(오늘) + 수동 변경 가능 (지난 날짜 소급 기록).

## 6. 사용자 시나리오

### 6.1 시나리오 1: 아침 등교 후 첫 화면

1. 김선생이 교실 PC를 켠다.
2. 쌤핀이 자동으로 실행된다 (시작 프로그램 등록).
3. **행사 팝업**: "D-3 중간고사" 알림이 뜬다. "확인" 클릭.
4. 메인 대시보드가 표시된다:
   - 상단: `3월 12일 (수) 08:35`, 날씨 `☁️ 5°C~12°C`, 미세먼지 `보통`
   - 메시지: "이번 주 수행평가 기간입니다!"
   - 시간표: 오늘(수) 열 강조, 아직 1교시 전이라 하이라이트 없음
   - 일정: 이번 달 학교 행사 리스트
5. 8:50이 되면 1교시 셀이 자동으로 노란색 하이라이트.
6. 학생들이 프로젝터 화면을 보고 오늘 시간표를 확인.

### 6.2 시나리오 2: 시간표 변경

1. 김선생이 사이드바에서 "시간표" 클릭.
2. "학급 시간표" 탭에서 "편집" 버튼 클릭.
3. 수요일 3교시를 "영어" → "국어"로 수정.
4. "저장" 클릭. 대시보드에 즉시 반영.

### 6.3 시나리오 3: 좌석 재배치

1. 월초 좌석 변경일. 김선생이 "좌석배치" 페이지로 이동.
2. 학생 A(1번)의 좌석을 드래그하여 학생 B(15번) 자리에 놓음.
3. 두 학생의 자리가 교환됨.
4. 여러 학생 자리를 드래그앤드롭으로 재배치.
5. 결과를 Excel로 내보내기 → 출력하여 게시.

### 6.4 시나리오 4: 나무학교 일정 확인

1. 박선생이 "일정" 페이지에서 "나무학교" 탭 클릭.
2. 4월의 나무학교 모임 날짜 확인.
3. 학교 일정 탭으로 전환하여 같은 날 학교 행사 없는지 크로스체크.
4. 새 나무학교 일정을 추가: 날짜 선택, 제목 입력, 추가.

### 6.5 시나리오 5: 수업 끝, 할일 정리

1. 6교시 수업 종료 후 김선생이 대시보드를 확인.
2. "오늘 할일" 위젯에서 "수행평가 기준표 작성" 완료 체크.
3. "메모" 페이지로 이동, 분홍 포스트잇에 "내일 교과 협의회 14:30" 메모 추가.
4. "오늘의 메시지"를 클릭하여 "내일은 교과 협의회가 있습니다."로 변경.

### 7.3 자동 업데이트

- electron-updater 기반
- GitHub Releases에서 latest 버전 자동 확인
- 업데이트 가능 시 알림 → 사용자 승인 후 다운로드 → 재시작 시 적용

## 10. 제약 조건 및 가정

### 제약 조건

1. **오프라인 우선**: 학교 네트워크가 불안정할 수 있음. 핵심 기능(시간표, 좌석, 일정, 메모)은 완전 오프라인 동작.
2. **Windows 우선**: 학교 컴퓨터는 거의 100% Windows. macOS는 Phase 3 이후.
3. **로컬 데이터**: 클라우드 동기화 없음. 데이터는 해당 PC에만 저장. 백업/복원은 JSON 파일 수동 관리.
4. **1인 개발**: 복잡한 백엔드나 서버 인프라 없이 순수 클라이언트 앱.

### 가정

1. 교사들은 PC 바탕화면에 쌤핀을 항상 띄워두고 사용한다.
2. 교실 PC에 관리자 권한으로 프로그램 설치 가능하다.
3. 시간표, 좌석, 일정 데이터는 교사가 직접 입력한다 (자동 연동 없음).
4. 학교별 교시 시간표가 다를 수 있으므로 반드시 설정 가능해야 한다.
