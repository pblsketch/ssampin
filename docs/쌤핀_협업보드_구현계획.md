# 쌤핀(SsamPin) 협업 온라인 보드 구현 계획서

> **작성일**: 2026-04-19
> **대상**: 쌤핀 v1.10.x 쌤도구 확장
> **목표**: 서버리스 잼보드 대안 - 교사 PC가 서버 역할

---

## 1. 프로젝트 개요

### 1.1 목표
Google Jamboard(2024.12 서비스 종료) 대안으로, 쌤핀 쌤도구에 **서버리스 실시간 협업 화이트보드**를 추가한다.

### 1.2 핵심 제약조건
- **서버리스**: 외부 서버 없이 교사 PC(Electron 앱)가 임시 서버 역할
- **개인정보 보호**: 학생 데이터가 교사 PC 로컬에만 저장
- **서버비 제로**: 유료 서버/SaaS 의존 없음
- **교사 제어**: 교사가 보드의 모든 권한을 가짐

### 1.3 기존 인프라 활용
| 기존 의존성 | 활용 목적 |
|---|---|
| `fabric` v7.2.0 | 기존 Canvas 렌더링 (좌석배치 등에 사용 중) |
| `ws` v8.19.0 | WebSocket 서버/클라이언트 |
| `cloudflared` v0.7.1 | Cloudflare 터널 (외부 접근) |
| `qrcode` v1.5.4 | QR 코드 생성 (접속 링크 공유) |
| `idb` v8.0.3 | IndexedDB (로컬 영속 저장) |
| `@supabase/supabase-js` | 선택적 클라우드 백업 |
| `zustand` v5.0.3 | 상태 관리 |

## 2. 기술 아키텍처

### 2.1 전체 구조

```
[교사 Electron 앱] ─────────────────────────────────
│
├─ Renderer Process (React)
│   ├─ Excalidraw 컴포넌트 (교사 보드 UI)
│   ├─ Y.js Document (CRDT 상태)
│   ├─ y-indexeddb (로컬 영속화)
│   └─ WebsocketProvider (Y.js 클라이언트)
│
├─ Main Process (Electron/Node.js)
│   ├─ Y.js WebSocket 서버 (@y/websocket-server 또는 직접 구현)
│   ├─ HTTP 정적 파일 서버 (학생용 웹 앱 서빙)
│   ├─ cloudflared 터널 관리
│   └─ 보드 데이터 로컬 파일 저장 (Y.encodeStateAsUpdate)
│
└─ QR 코드 → 터널 URL 공유

[학생 브라우저 (모바일/PC)] ─────────────────────────
│
├─ 경량 웹 앱 (Excalidraw + Y.js 클라이언트)
├─ WebsocketProvider → cloudflared 터널 → 교사 PC
└─ viewModeEnabled 또는 제한된 편집 모드
```

### 2.2 데이터 흐름

```
1. 교사: "협업 보드 열기" 클릭
2. Electron Main: WebSocket 서버 시작 (포트 자동 선택)
3. Electron Main: cloudflared 터널 생성 → 공개 URL 획득
4. Electron Renderer: QR 코드 표시 (터널 URL)
5. 학생: QR 스캔 → 브라우저에서 웹 앱 로드
6. 학생 웹 앱: Y.js WebsocketProvider로 교사 PC에 연결
7. 실시간 CRDT 동기화: 모든 드로잉이 자동 병합
8. 교사: 보드 닫기 → 터널 종료, 데이터 로컬 저장
```

## 3. 핵심 기술 스택 결정

### 3.1 캔버스 엔진: Excalidraw (확정)

**선택 이유**:
- MIT 라이선스 (GPL-3.0 호환)
- React 18 컴포넌트로 즉시 임베드
- y-excalidraw 바인딩으로 Y.js CRDT 연동 검증됨
- 풍부한 내보내기 (PNG, SVG, JSON)
- viewModeEnabled, UIOptions 등 교사 제어에 필요한 props 완비
- **Jamboard 스타일 커스터마이징 가능** (아래 3.4절 참고)

**설치**:
```bash
npm install @excalidraw/excalidraw yjs y-websocket y-indexeddb y-excalidraw
```

**Vite 설정 필수**:
```ts
// vite.config.ts
export default defineConfig({
  optimizeDeps: {
    esbuildOptions: {
      target: "es2022",  // Excalidraw v0.18+ 필수
    },
  },
})
```

**Electron 에셋 설정**:
```ts
// Excalidraw 폰트를 로컬에서 서빙 (CDN 의존 제거)
window.EXCALIDRAW_ASSET_PATH = "/excalidraw-assets/";
// node_modules/@excalidraw/excalidraw/dist/prod/fonts → public/excalidraw-assets/
```

### 3.2 실시간 동기화: Y.js + y-websocket (확정)

**선택 이유**:
- CRDT 기반 → 충돌 없는 자동 병합
- y-websocket: 교사 PC에서 WebSocket 서버 구동 가능
- 30명 동시 접속 충분 (Y.js는 수백 명까지 테스트됨)
- y-indexeddb로 오프라인/영속 저장

**y-webrtc 미사용 이유**:
- 시그널링 서버 필요 (외부 의존)
- P2P 메시 → 30명이면 연결 수 폭발적 증가
- WebSocket(교사 PC 중계) 모델이 더 안정적

### 3.3 Cloudflare 터널 WebSocket 호환성

**확인된 사실**:
- Cloudflare는 모든 플랜에서 WebSocket 프록시 지원
- cloudflared 터널을 통한 ws:// 연결은 자동으로 wss://로 업그레이드됨
- 일부 브라우저(Firefox HTTPS-Only 모드)에서 ws→wss 업그레이드 이슈 보고됨
  → **해결**: 터널은 기본적으로 HTTPS이므로 wss:// 직접 사용

### 3.4 Jamboard 스타일 UI 커스터마이징

Excalidraw의 기본 UI는 손그림 스케치 스타일로 Jamboard와 차이가 크다. 아래 전략으로 Jamboard에 가까운 교육용 UI를 구현한다.

#### 3.4.1 Excalidraw vs Jamboard 핵심 차이

```
                 Excalidraw (기본)              Jamboard
시각 스타일      손그림(스케치) 느낌             깔끔한 직선/도형
툴바 위치        상단 가로 (10개+ 도구)          좌측 세로 (6개 도구)
색상 팔레트      풍부한 색상 피커               심플한 6색
캔버스 구조      무한 캔버스                    페이지(프레임) 기반
스티키 노트      내장 없음 (텍스트+도형 조합)     1등 기능
```

#### 3.4.2 깔끔한 직선 스타일 (roughness: 0)

**난이도: 쉬움 / 달성도: 90%**

Excalidraw의 `roughness` 속성을 0으로 설정하면 "Architect 모드"가 적용되어 깔끔한 직선과 도형이 그려진다.

```tsx
<Excalidraw
  initialData={{
    appState: {
      currentItemRoughness: 0,         // 깔끔한 직선 (핵심)
      currentItemRoundness: "sharp",   // 각진 코너
      currentItemStrokeStyle: "solid", // 실선
      currentItemFontFamily: 4,        // 시스템 폰트 (손글씨 아님)
    }
  }}
/>
```

**강제 적용 전략**: 사용자가 속성을 수동으로 바꾸는 것을 방지하기 위해 `onChange` 콜백에서 roughness를 감시하고, 학생 모드에서는 항상 0으로 강제 복원한다.

```tsx
const handleChange = useCallback((elements, appState) => {
  // 학생 모드에서 roughness를 항상 0으로 강제
  if (!isTeacher && appState.currentItemRoughness !== 0) {
    excalidrawAPI?.updateScene({
      appState: { currentItemRoughness: 0 },
    });
  }
}, [isTeacher, excalidrawAPI]);
```

#### 3.4.3 Jamboard 스타일 커스텀 툴바

**난이도: 중간 / 달성도: 80%**

Excalidraw의 메인 도구 툴바를 CSS로 숨기고, Jamboard처럼 단순한 커스텀 툴바를 React로 구현한다.

```css
/* Excalidraw 기본 도구 툴바 숨기기 */
.ssampin-board .excalidraw .App-toolbar {
  display: none !important;
}
/* 속성 패널도 학생에게는 숨기기 (선택) */
.ssampin-board.student-mode .excalidraw .App-menu {
  display: none !important;
}
```

```tsx
// Jamboard 스타일 좌측 세로 툴바
function JamboardToolbar({ api, isTeacher }: { api: any; isTeacher: boolean }) {
  const tools = [
    { icon: "✏️", type: "freedraw", label: "펜" },
    { icon: "⬜", type: "rectangle", label: "도형" },
    { icon: "T",  type: "text", label: "텍스트" },
    { icon: "📌", action: "stickyNote", label: "스티키 노트" },
    { icon: "🧹", type: "eraser", label: "지우개" },
    { icon: "👆", type: "selection", label: "선택" },
  ];
  // 교사 전용 도구 추가
  if (isTeacher) {
    tools.push(
      { icon: "🖼️", type: "image", label: "이미지" },
      { icon: "🔴", action: "laser", label: "레이저" },
    );
  }
  return (
    <div className="jamboard-toolbar"> {/* 좌측 세로 배치 */}
      {tools.map((t) => (
        <button
          key={t.type || t.action}
          className="jamboard-tool-btn"
          title={t.label}
          onClick={() => {
            if (t.action === "stickyNote") addStickyNote(api);
            else api.setActiveTool({ type: t.type });
          }}
        >
          {t.icon}
        </button>
      ))}
    </div>
  );
}
```

**주의**: CSS로 `.App-toolbar`를 숨기는 방식은 Excalidraw 버전 업데이트 시 클래스명이 변경될 수 있어 fragile하다. 업데이트 시 CSS 선택자 점검 필요.

#### 3.4.4 스티키 노트 기능

**난이도: 쉬움 / 달성도: 95%**

Excalidraw의 `convertToExcalidrawElements` API로 색상 배경 사각형 + 텍스트 조합의 스티키 노트를 프로그래밍적으로 생성한다.

```tsx
import { convertToExcalidrawElements } from "@excalidraw/excalidraw";
import { CaptureUpdateAction } from "@excalidraw/excalidraw";

// 스티키 노트 색상 옵션 (Jamboard와 동일한 6색)
const STICKY_COLORS = [
  { bg: "#fff3bf", stroke: "#e8c800", label: "노랑" },
  { bg: "#d0f0c0", stroke: "#5cb85c", label: "초록" },
  { bg: "#c4e1ff", stroke: "#4a90d9", label: "파랑" },
  { bg: "#ffd6e0", stroke: "#e85d75", label: "분홍" },
  { bg: "#e8daef", stroke: "#9b59b6", label: "보라" },
  { bg: "#fce5cd", stroke: "#e67e22", label: "주황" },
];

function addStickyNote(api: any, colorIndex = 0) {
  const color = STICKY_COLORS[colorIndex];
  const { scrollX, scrollY, width, height } = api.getAppState();
  // 화면 중앙에 배치
  const centerX = -scrollX + width / 2 - 100;
  const centerY = -scrollY + height / 2 - 75;

  const stickyNote = convertToExcalidrawElements([
    {
      type: "rectangle",
      x: centerX,
      y: centerY,
      width: 200,
      height: 150,
      backgroundColor: color.bg,
      strokeColor: color.stroke,
      fillStyle: "solid",
      roughness: 0,                  // 깔끔한 스타일
      roundness: { type: 3 },        // 약간 둥근 코너
      strokeWidth: 1,
      label: {
        text: "메모를 입력하세요",
        fontSize: 16,
        textAlign: "center",
        verticalAlign: "middle",
      },
    },
  ]);

  api.updateScene({
    elements: [...api.getSceneElements(), ...stickyNote],
    captureUpdate: CaptureUpdateAction.IMMEDIATELY,
  });
}
```

#### 3.4.5 페이지(프레임) 전환 시스템

**난이도: 중간 / 달성도: 85%**

Excalidraw는 무한 캔버스이므로 Jamboard의 "프레임(페이지)" 개념을 Scene 직렬화로 직접 구현한다.

```tsx
import { serializeAsJSON, loadFromBlob } from "@excalidraw/excalidraw";

interface BoardPage {
  id: string;
  name: string;
  data: string; // serializeAsJSON 결과
  thumbnail?: string; // 미리보기 이미지 URL
}

function usePageSystem(api: any) {
  const [pages, setPages] = useState<BoardPage[]>([
    { id: "page-1", name: "페이지 1", data: "" },
  ]);
  const [currentPageIndex, setCurrentPageIndex] = useState(0);

  // 현재 페이지 저장
  const saveCurrentPage = () => {
    const json = serializeAsJSON(
      api.getSceneElements(),
      api.getAppState(),
      api.getFiles(),
      "local"
    );
    setPages((prev) => {
      const updated = [...prev];
      updated[currentPageIndex] = { ...updated[currentPageIndex], data: json };
      return updated;
    });
  };

  // 다른 페이지로 전환
  const switchToPage = async (targetIndex: number) => {
    saveCurrentPage();
    const targetPage = pages[targetIndex];
    if (targetPage.data) {
      const blob = new Blob([targetPage.data], { type: "application/json" });
      const { elements, appState } = await loadFromBlob(blob, null, null);
      api.updateScene({ elements, appState });
    } else {
      api.resetScene(); // 빈 페이지
    }
    setCurrentPageIndex(targetIndex);
  };

  // 새 페이지 추가
  const addPage = () => {
    saveCurrentPage();
    const newPage = { id: `page-${pages.length + 1}`, name: `페이지 ${pages.length + 1}`, data: "" };
    setPages((prev) => [...prev, newPage]);
    api.resetScene();
    setCurrentPageIndex(pages.length);
  };

  return { pages, currentPageIndex, switchToPage, addPage, saveCurrentPage };
}
```

하단에 Jamboard 스타일 페이지 네비게이터 UI:
```tsx
function PageNavigator({ pages, currentIndex, onSwitch, onAdd }) {
  return (
    <div className="page-navigator"> {/* 하단 가로 스크롤 */}
      {pages.map((page, i) => (
        <button
          key={page.id}
          className={`page-thumb ${i === currentIndex ? "active" : ""}`}
          onClick={() => onSwitch(i)}
        >
          {page.thumbnail ? <img src={page.thumbnail} /> : <span>{i + 1}</span>}
        </button>
      ))}
      <button className="page-add" onClick={onAdd}>+</button>
    </div>
  );
}
```

#### 3.4.6 색상 팔레트 단순화

**난이도: 중간 / 달성도: 70%**

Excalidraw의 기본 색상 피커를 API로 교체하는 방법은 공식 지원되지 않는다. 커스텀 컬러 팔레트를 만들어 `excalidrawAPI.updateScene`으로 색상을 적용하는 방식을 사용한다.

```tsx
// 단순화된 Jamboard 스타일 6색 팔레트
const SIMPLE_COLORS = ["#1e1e1e", "#e03131", "#2f9e44", "#1971c2", "#f08c00", "#9c36b5"];

function SimpleColorPicker({ api }) {
  return (
    <div className="simple-color-picker">
      {SIMPLE_COLORS.map((color) => (
        <button
          key={color}
          style={{ backgroundColor: color }}
          onClick={() => api.updateScene({
            appState: {
              currentItemStrokeColor: color,
              currentItemBackgroundColor: color,
            }
          })}
        />
      ))}
    </div>
  );
}
```

기본 색상 피커는 CSS로 숨기기 가능하나 fragile:
```css
.ssampin-board .excalidraw .color-picker-container { display: none !important; }
```

#### 3.4.7 CSS 테마 및 배경

**난이도: 쉬움 / 달성도: 95%**

Excalidraw는 CSS 변수 오버라이드를 공식 지원한다.

```css
/* Jamboard 느낌의 깔끔한 UI 테마 */
.ssampin-board .excalidraw {
  --color-primary: #4285f4;          /* Google Blue */
  --color-primary-darker: #3367d6;
  --color-primary-darkest: #2851a3;
  --color-primary-light: #d2e3fc;
  --border-radius-md: 8px;
  --border-radius-lg: 12px;
}

/* 배경색 */
.ssampin-board .excalidraw {
  --island-bg-color: #ffffff;
}
```

배경색과 그리드는 prop으로 직접 제어:
```tsx
<Excalidraw
  theme="light"
  gridModeEnabled={false}
  initialData={{
    appState: {
      viewBackgroundColor: "#ffffff", // 흰 배경 (Jamboard 기본)
    }
  }}
  UIOptions={{
    canvasActions: {
      changeViewBackgroundColor: false, // 배경색 변경 UI 숨기기
      toggleTheme: false,               // 다크모드 토글 숨기기
    }
  }}
/>
```

### 4.2 보드 모드 (3가지)

#### 모드 A: 공유 보드 (Shared Canvas)
- 잼보드 스타일: 하나의 큰 캔버스에 모두가 동시 편집
- Y.js 단일 room으로 동기화
- 교사: 모든 도구 사용, 학생: 제한된 도구

#### 모드 B: 개인 보드 (Individual Boards)  
- Whiteboard.fi 스타일: 학생마다 독립 보드
- 교사 대시보드에서 30개 보드 그리드 뷰로 모니터링
- 학생은 자기 보드만 편집, 타 학생 보드 미공개

#### 모드 C: 그룹 보드 (Group Boards)
- 교사가 5~6명 그룹 편성 → 그룹별 공유 보드
- Miro의 Frames 패턴: 무한 캔버스 위에 그룹별 영역 할당

### 4.3 교사 제어 기능

```
교사 제어판
├─ 보드 잠금/해제 (학생 편집 일시 중지)

├─ 포커스 모드 (학생 화면을 교사 보드로 강제 전환)
```


### 9.2 Y.js WebSocket 서버 (Electron Main)

```ts
// electron/board-server.ts
import { WebSocketServer } from "ws";
import * as Y from "yjs";

interface BoardRoom {
  doc: Y.Doc;
  clients: Set<WebSocket>;
}

const rooms = new Map<string, BoardRoom>();

export function startBoardServer(port: number): WebSocketServer {
  const wss = new WebSocketServer({ port });

  wss.on("connection", (ws, req) => {
    const roomName = new URL(req.url!, `http://localhost:${port}`).searchParams.get("room") || "default";

    if (!rooms.has(roomName)) {
      rooms.set(roomName, { doc: new Y.Doc(), clients: new Set() });
    }

    const room = rooms.get(roomName)!;
    room.clients.add(ws as any);

    // 초기 상태 전송
    const state = Y.encodeStateAsUpdate(room.doc);
    ws.send(state);

    ws.on("message", (data: Buffer) => {
      Y.applyUpdate(room.doc, new Uint8Array(data));
      // 다른 클라이언트에게 브로드캐스트
      room.clients.forEach((client) => {
        if (client !== ws && (client as any).readyState === 1) {
          (client as any).send(data);
        }
      });
    });

    ws.on("close", () => {
      room.clients.delete(ws as any);
    });
  });

  return wss;
}
```

## 11. 결론

쌤핀은 이미 협업 보드에 필요한 핵심 인프라(`ws`, `cloudflared`, `qrcode`, `idb`)를 갖추고 있으며, **Excalidraw + Y.js** 조합으로 서버리스 실시간 협업 보드를 구현할 수 있다.

**아키텍처 핵심**: 교사 Electron 앱이 Y.js WebSocket 서버 + 정적 파일 서버 역할을 하고, cloudflared 터널로 학생이 외부에서 접속. 모든 데이터는 교사 PC 로컬에만 저장되어 개인정보 보호와 서버비 제로를 동시에 달성.

**Jamboard 스타일 UI**: Excalidraw의 roughness:0 + 커스텀 좌측 툴바 + 스티키 노트 + 페이지 시스템 + CSS 테마 오버라이드로 **Jamboard 80~90% 수준의 UX**를 달성할 수 있다. 하이브리드 방식(Excalidraw 캔버스 엔진 70% + React 커스텀 UI 30%)이 공수 대비 효과가 가장 좋다.

**기존 패턴 재사용**: 좌석배치(dnd-kit), 학생명단(Zustand store), QR 공유(qrcode), 터널(cloudflared), 내보내기(exceljs/hwpxcore) 등 쌤핀의 기존 인프라와 자연스럽게 통합 가능.
