# 페이지 헤더 통일 계획 — 쌤핀

> 2026-04-24 작성. 12개 페이지(대시보드 제외) 헤더 디자인 불일치 해소.
> 작성: bkit:frontend-architect 서브에이전트 분석.

---

## 0. 배경

현재 쌤핀의 각 페이지별 헤더 스타일이 제각각 — 제목 크기(xl/2xl/3xl 혼재), font-weight(`font-bold`/`font-black`), 구분선 유무, padding 방식(`px-8 py-4`/`h-16`/`h-20`/`mb-6` 등 8가지), 헤더 태그(`<header>`/`<div>`) 등. 사용자가 통일 요청.

## 1. 현황 매트릭스 (12개 페이지)

| 페이지 | 제목 태그/크기 | weight | 이모지 | 구분선 | padding | 문제 |
|--------|---------------|--------|--------|--------|---------|------|
| 일정 관리 | `<h2> text-xl xl:text-2xl` | `font-bold` | `<span>` | ✅ | `px-8 py-4` | 표준에 가장 근접. weight가 sp 토큰 아님 |
| 시간표 | `<h2> text-3xl` | `font-black` | `<span>` + `<p>` 서브타이틀 | ❌ | `pb-6` | 크기 과다, border-b 없음 |
| 좌석 배치 | `<h2> text-xl` | `font-bold` | material icon + bg 박스 | ❌ | `pb-6` | border-b 없음. 아이콘 래핑 이질적 |
| 담임 업무 | `<h2> text-2xl` | `font-bold` | `<span>` | ❌ | `mb-6` | `<div>` 사용. border-b 없음 |
| 메모 | `<h2> text-2xl` | `font-black` | material `text-[28px]` | ✅ | `h-16 px-8` | 고정 h-16 강제. `font-black` |
| 쌤핀 노트 | `<h2> text-2xl` | `font-bold` | `<span>` | ❌ | `gap-4` | padding 없이 gap만 |
| 할 일 | `<h2> text-2xl` | `font-bold` | `<span> text-3xl` | ✅ | `h-20 px-8` | 진행률 바가 헤더 오염 |
| 수업 관리 | `<h1> text-xl` | `font-bold` | material `text-2xl` | ❌ | `mb-6` | `<div>` 사용. h1/h2 혼용 |
| 급식 | `<h2> text-3xl` | `font-black` | `<span> text-2xl` | ⚠️ `/30` | `px-8 py-6` + sticky | 크기 과다, 불투명도 30%만 |
| 설정 | `<h2> text-2xl` | `font-black` | material `muted` | ✅ | `px-8 py-5` | `font-black`, py 불일치 |
| 내보내기 | 없음 (콘텐츠 블록) | `font-bold` | material + bg 박스 | ❌ | `mb-8` | `<header>` 자체 없음 |
| 도구 | `<h1> text-2xl` | `font-bold` | material `text-[28px]` | ❌ | `mb-8` | `<div>` 사용 |

### 5대 불일치
1. 제목 크기 `xl` / `2xl` / `3xl` 혼재
2. `font-bold` / `font-black` 혼용, `font-sp-bold` 사용 없음
3. 구분선 5 / 7 페이지 분열
4. 헤더 태그 `<header>` 8개, `<div>` 4개, 없음 1개
5. padding 방식 8가지 혼재

## 2. 표준 헤더 스펙

### 2-1. 컨테이너
```
<header className="shrink-0 px-8 py-4 flex flex-wrap items-center gap-3 border-b border-sp-border bg-sp-bg">
```
- 태그: `<header>` 시맨틱 고정 (div 금지)
- padding: `px-8 py-4` 전 페이지 동일
- 구분선: `border-b border-sp-border` 전 페이지 적용
- **고정 높이 `h-16`/`h-20` 금지** — padding만 사용

### 2-2. 제목
```
<h1 className="text-xl xl:text-2xl font-sp-bold text-sp-text flex items-center gap-2">
  <span className="text-2xl xl:text-3xl" aria-hidden="true">🗂️</span>
  페이지 제목
</h1>
```
- 태그 `<h1>` 통일 (각 페이지가 독립 SPA 뷰)
- 크기 `text-xl xl:text-2xl` (20→24px)
- weight **`font-sp-bold` 필수** (Pretendard 680 axis) — `font-bold`/`font-black` 금지
- 이모지 `aria-hidden="true"` 추가
- material icon 사용 시 `text-[26px] xl:text-[30px]`, 배경 박스 패턴 폐기

### 2-3. 뷰 토글 (leftAddon 슬롯)
Schedule.tsx pill 그룹을 표준 채택:
```
<div className="flex items-center bg-sp-surface/60 rounded-lg p-0.5 border border-sp-border gap-0.5">
  <button className="px-3 xl:px-4 py-1.5 rounded-md text-xs xl:text-sm transition-all duration-sp-base ease-sp-out
    [active] bg-sp-card shadow-sp-sm font-sp-semibold text-sp-text
    [inactive] font-sp-medium text-sp-muted hover:text-sp-text">
```

### 2-4. 우측 액션 버튼
- radius: **`rounded-xl`** (기존 `rounded-lg` 일괄 교체)
- `px-3 xl:px-4 py-2 xl:py-2.5`
- `text-xs xl:text-sm font-sp-semibold`
- 레이블 반응형 `hidden sm:inline`
- transition `duration-sp-base ease-sp-out active:scale-95`
- secondary: `border border-sp-border text-sp-muted hover:text-sp-text hover:bg-sp-surface`
- primary: `bg-sp-accent text-white hover:brightness-110 shadow-sp-accent`

### 2-5. Sticky 정책
- 콘텐츠 스크롤 페이지만 적용 (급식 등)
- 적용 시: `sticky top-0 bg-sp-bg/95 backdrop-blur-sm z-10`
- 부모의 `overflow: hidden` 사전 확인 필수

## 3. `<PageHeader>` 신규 컴포넌트

경로: `src/adapters/components/common/PageHeader/PageHeader.tsx`

### Props
```ts
interface PageHeaderProps {
  icon?: string;
  iconIsMaterial?: boolean;  // true = material-symbols, false = 이모지 (기본)
  title: string;
  leftAddon?: ReactNode;     // 뷰 토글, 탭 등
  rightActions?: ReactNode;  // 우측 버튼 그룹
  sticky?: boolean;
}
```

### 구현
```tsx
export function PageHeader({ icon, iconIsMaterial = false, title, leftAddon, rightActions, sticky = false }: PageHeaderProps) {
  const stickyCls = sticky ? 'sticky top-0 bg-sp-bg/95 backdrop-blur-sm z-10' : 'bg-sp-bg';
  return (
    <header className={`shrink-0 px-8 py-4 flex flex-wrap items-center gap-3 border-b border-sp-border ${stickyCls}`}>
      <div className="flex items-center gap-4 mr-auto">
        <h1 className="text-xl xl:text-2xl font-sp-bold text-sp-text flex items-center gap-2">
          {icon && (iconIsMaterial
            ? <span className="material-symbols-outlined text-[26px] xl:text-[30px] text-sp-muted" aria-hidden="true">{icon}</span>
            : <span className="text-2xl xl:text-3xl" aria-hidden="true">{icon}</span>
          )}
          {title}
        </h1>
        {leftAddon}
      </div>
      {rightActions && <div className="flex items-center gap-1.5 xl:gap-2 flex-wrap">{rightActions}</div>}
    </header>
  );
}
```

## 6. 리스크 & 주의사항

1. **진행률 바(Todo)·컬러피커(Memo)** 같은 비-버튼 UI는 rightActions에 넣어도 어색할 수 있음. 별도 행으로 분리 고려
2. **반응형 breakpoint 혼선** — `md:` 등 혼재된 기존 클래스 제거. 표준은 `sm:`(레이블 숨김) + `xl:`(크기 확대) 2개만
3. **Sidebar collapsed 영향** — `flex-wrap`과 `hidden sm:inline`으로 자연 축소 처리
4. **sticky 주의** — 부모의 `overflow: hidden` 사전 확인. 쌤핀의 `-m-8 flex flex-col h-[calc(100%+4rem)]` 패턴 주의
5. **`rounded-sp-*` 절대 금지** — 이전 정책(feedback_rounding_policy) 재확인
6. **`font-sp-bold` 토큰 존재 확인** — tailwind.config.js 등록 여부 Phase 1 전 확인
