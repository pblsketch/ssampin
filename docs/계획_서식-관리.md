# 쌤핀 서식 관리 (HWPX + PDF + Excel 통합) — 구현 계획서

> **작성일**: 2026-04-20
> **대상 버전**: v1.11.0 (Phase 1 MVP) ~ v1.13.0 (mail-merge 완성)
> **선행 조건**: `PdfExporter.ts`가 `pdf-lib` 기반으로 재작성되어 있을 것 (별도 계획서 `docs/계획_PDF-Exporter.md`)
> **차별화 축**: 위노트(WeeNote) PDF 드로워 대비 **HWPX + PDF + Excel 3종 통합 + 학생 mail-merge + 학급 일괄 출력**

---

## 1. 개요 & 위노트 대비 차별화

### 1-1. 한 줄 요약
쌤핀은 한국 교사가 실제 쓰는 3종 포맷(HWPX/PDF/Excel)을 **한 자리에 모으고**, 학생 데이터를 `{{학생이름}}` 같은 **자동 채움**으로 녹여서, **학급 29명치 가정통신문을 1-클릭으로 뽑게** 한다.

### 1-2. 경쟁 비교표
| 축 | 위노트(WeeNote) | 쌤핀(목표) |
|---|---|---|
| 지원 포맷 | PDF만 | **HWPX + PDF + Excel** |
| 등록 방식 | 파일 업로드 | 파일 업로드 + 기존 Export 결과 저장 + 기본 내장 10종 |
| 카테고리 | 불명확 | 6개 기본(가정통신문/공문서/성적평가/생활기록/상담/기타) + 사용자 추가 |
| 미리보기 | 단순 표시 | **첫 페이지 썸네일 자동 생성** |
| 학생 데이터 채움 | ❌ | ✅ `{{placeholder}}` → 학급 전체 치환 |
| 일괄 출력 | ❌ | ✅ 29명 → 29개 파일 또는 1개 통합 PDF |
| 즐겨찾기/최근사용 | 불명확 | ⭐ 즐겨찾기 + `lastUsedAt` 정렬 |
| 바로 인쇄 | PDF만 | 3종 모두 (Electron `printToPDF` + OS 프린트 대화상자) |

### 1-3. 이 계획서의 범위
- **In-scope**: 서식 CRUD · 카테고리/필터/검색 · 썸네일 · mail-merge · 학급 일괄 출력 · 기본 내장 10종
- **Out-of-scope**: PDF 렌더링 구현(선행 `계획_PDF-Exporter.md`가 책임), 쌤핀 노트 연동(Phase 4로 분리)

## 2. 사용자 시나리오

### 시나리오 A — 학부모 상담 안내문 29명 일괄 출력
1. 교사 박선생, 4월 둘째 주에 **학부모 상담 주간** 진행 예정
2. 사이드바 > "서식" 클릭 → `가정통신문` 카테고리 필터
3. **"학부모 상담 신청 안내문"** 카드 클릭 (내장 서식, HWPX)
4. 미리보기 모달 → "학생 데이터 채우기" 클릭
5. 채움 마법사:
   - Step 1: 대상 선택 (`전체 학급` / `특정 학생`)
   - Step 2: 출력 모드 (`개별 파일 29개` / `통합 PDF 29페이지`)
   - Step 3: 커스텀 필드 입력(`{{상담기간}}` = "2026-04-14 ~ 04-18")
6. "출력" 클릭 → 학급 저장 폴더에 `상담안내_홍길동.hwpx` … `상담안내_김철수.hwpx` 29개 생성
7. 토스트: "29개 파일 생성 완료 · [폴더 열기] [모두 인쇄]"

### 시나리오 B — 기존 Excel 양식 업로드
1. 교사가 학교에서 받은 `수행평가_채점표.xlsx`를 보유
2. 서식 관리 > "+ 서식 등록" → 파일 드래그
3. 쌤핀이 Excel 셀을 훑으며 `{{...}}` 패턴 자동 감지 → `{{학생이름}}`, `{{번호}}` 발견
4. 카테고리 `성적/평가` 자동 제안 + 썸네일 자동 생성
5. 저장 → 카드 목록에 등장, 다음부터 일괄 채움 가능

### 시나리오 C — 기본 내장 서식 즉시 사용
1. 쌤핀 첫 설치 시 `내장 서식 10종`이 자동 등록됨(읽기 전용)
2. 교사가 그대로 사용하거나 "복사본 수정"으로 커스텀
3. 복사본은 사용자 서식으로 분리되어 저장 — 원본 10종은 항상 유지

## 3. 라이브러리 조사 결과

### 3-1. PDF 템플릿 엔진 비교

| 라이브러리 | 방식 | 한글 폰트 | 쌤핀 적합도 | 메모 |
|---|---|---|---|---|
| **pdf-lib** ([github](https://github.com/Hopding/pdf-lib)) | AcroForm 폼 필드 fill + 좌표 기반 | fontkit + TTF 임베딩 필요 | ⭐⭐⭐⭐⭐ | PDF Exporter에서 이미 채택 예정. `getForm().getTextField('name').setText(value)` 패턴. 한글은 Noto Sans KR 서브셋 임베딩. |
| **pdfme** v5 ([github](https://github.com/pdfme/pdfme)) | JSON 스키마 + WYSIWYG 디자이너 | CJK 폰트 embed/subset 지원 공식 문서화 | ⭐⭐⭐⭐ | 쌤핀 **기본 내장 서식 제작용**으로 우수. designer로 템플릿 만들고 generator로 주입. AcroForm 직접 편집은 미지원이지만 사용자 업로드 PDF에는 쌤핀 자체 placeholder 파서가 더 유연. |
| **pdfmake** ([npm](https://www.npmjs.com/package/pdfmake)) | 선언적 JSON(문서 정의서) | CJK는 `.otf` base64 임베딩, merge 불가 이슈 존재 | ⭐⭐ | 템플릿 기반 아니라 코드로 생성. 업로드 PDF 채움에는 부적합. |
| **pdf-fill-form** | AcroForm fill (Node, native dep) | 제한적 | ⭐ | native 바이너리 의존, Electron 크로스플랫폼에 리스크. |
| **carbone.io** ([site](https://carbone.io/)) | Template + data → PDF/DOCX/XLSX 범용 | 다국어 문서상 지원 | ⭐⭐ | **160+ 포맷 변환** 강력하지만 LibreOffice 외부 바이너리 의존 → Electron 번들링 난이도 ↑. 서버형 성격. |

**결론**: PDF 쪽은 **pdf-lib가 주 엔진** (선행 계획서가 채택한 스택을 그대로 씀). 내장 서식 제작 시 pdfme 디자이너로 템플릿(JSON) 만들고 pdfme generator로 한번만 PDF를 굽는 방식이 선택적으로 유효. 사용자 업로드 PDF에 대한 mail-merge는 **pdf-lib + 쌤핀 자체 placeholder 파서** 조합이 가장 깔끔.

### 3-2. Excel 템플릿 엔진 비교

| 라이브러리 | 현재 사용 | 템플릿 치환 방식 | 쌤핀 적합도 |
|---|---|---|---|
| **exceljs** ([npm](https://www.npmjs.com/package/exceljs)) | ✅ 이미 사용 중 | `workbook.xlsx.readFile()` → `ws.eachRow((row) => row.eachCell(...))`로 순회하며 `cell.value`를 regex replace. Rich text는 `cell.value.richText` 배열 순회 필요. | ⭐⭐⭐⭐⭐ |
| **xlsx-populate** ([github](https://github.com/dtjohnson/xlsx-populate)) | 미사용 | `workbook.find("foo", "bar")` 한 줄로 전체 치환. defined names 지원. | ⭐⭐⭐ |
| **xlsx-template** ([npm](https://www.npmjs.com/package/xlsx-template)) | 미사용 | `{placeholder}` 문법, 배열→다중 행 자동 생성, 이미지 삽입 지원. | ⭐⭐⭐ |
| **sheetjs / xlsx** | 미사용 | 저수준 읽기/쓰기. 템플릿 치환은 자체 구현 필요. | ⭐⭐ |

**결론**: **exceljs 유지**. 이미 수많은 Export 경로가 exceljs로 돌고 있고, `cell.value`의 문자열/richText 양쪽을 모두 훑는 placeholder 파서를 쌤핀에서 한 번만 구현하면 충분. 새 의존성 추가 없음.

### 3-3. HWPX 템플릿/치환

| 라이브러리 | 현재 사용 | 치환 지원 | 메모 |
|---|---|---|---|
| **@ubermensch1218/hwpxcore** ([npm](https://www.npmjs.com/package/@ubermensch1218/hwpxcore)) | ✅ 이미 사용 중 | 공식 placeholder 치환 API는 **없음**. `doc.paragraphs`, `doc.tables` 순회 + 셀 텍스트 정규식 치환으로 우회 가능. [`HwpxExporter.ts`](../src/infrastructure/export/HwpxExporter.ts)가 `table.setCellText()` API를 이미 활용 중. | 읽기/쓰기 가능. OOXML 유사 DOM 제공. |
| **@ubermensch1218/hwpxeditor** ([npm](https://www.npmjs.com/package/@ubermensch1218/hwpxeditor)) | 미사용 | hwpxcore 기반 에디터 컴포넌트. 서식 직접 편집 UI가 필요하면 후보. | 이번 범위에는 불필요. |
| **@ssabrojs/hwpxjs** ([npm](https://www.npmjs.com/package/@ssabrojs/hwpxjs)) | 미사용 | `HwpxReader.extractText()` + JS `.replace()` 조합으로 `{{key}}` 치환 문서화됨. TypeScript 네이티브. | 보조 후보. hwpxcore가 저수준이라 mail-merge 편의성은 ssabrojs 쪽이 더 나을 가능성. |
| **hwp.js** ([github](https://github.com/hahnlee/hwp.js)) | 미사용 | HWP(구형) 뷰어·파서. HWPX 쓰기 미지원. | 부적합. |
| **hwpxlib (Java)** ([github](https://github.com/neolord0/hwpxlib)) | 미사용 | Java 전용. | 부적합. |

**결론**: **hwpxcore 유지가 기본**, 단 HWPX의 내부 XML 구조상 paragraph/run 분절로 `{{학생이름}}` 같은 토큰이 **여러 run에 걸쳐 쪼개질 위험**이 있음 (한글 편집기에서 입력 중 서식 변경이 일어나면 흔히 발생). 이에 대한 대응:
- **옵션 A (권장)**: hwpxcore의 모든 paragraph를 순회 → 같은 paragraph 내 run들의 텍스트를 concat → 치환 → 첫 run에만 결과 텍스트 주입, 나머지 run 비움. 구현 난이도 중.
- **옵션 B**: 치환 실패율이 높으면 **@ssabrojs/hwpxjs**를 병용하여 XML 문자열 레벨에서 `.replace(/{{학생이름}}/g, value)` 직접 적용 (단, 폰트/서식 깨질 위험 있어 사전 테스트 필요).

### 3-4. Mail-merge 범용 엔진

| 라이브러리 | 대상 포맷 | 쌤핀 채택? |
|---|---|---|
| **docxtemplater** ([docxtemplater.com](https://docxtemplater.com/)) | DOCX/PPTX/XLSX | ❌ (HWPX 미지원이 치명적. 단, Mustache 스타일 `{placeholder}` · loops · conditions 문법은 **쌤핀 자체 파서가 참고**) |
| **easy-template-x** ([github](https://github.com/alonrbar/easy-template-x)) | DOCX 전용 | ❌ 동일 이유 |
| **carbone.io** | 범용 | ❌ LibreOffice 외부 의존 |
| **쌤핀 자체 엔진** | HWPX + PDF + Excel | ✅ **채택** |

**결론**: 범용 라이브러리 하나로 3종을 다 커버할 수 없으므로, **쌤핀 자체 mail-merge 엔진**을 만들고 포맷별로 dispatch한다. 문법은 Mustache 스타일 `{{...}}` (docxtemplater 관례 존중). 루프·조건은 **Phase 3 이후로 지연** — MVP는 단순 scalar 치환만.

### 3-5. 썸네일 생성

| 포맷 | 방법 | 비용 |
|---|---|---|
| PDF | **pdfjs-dist** + offscreen canvas로 첫 페이지 렌더 → `toDataURL('image/png')` | 낮음. Electron renderer에서 곧바로 동작. [pdfjs-dist](https://www.npmjs.com/package/pdfjs-dist) |
| Excel | 시트 → canvas 렌더는 비현실적. **대안**: 첫 시트 상단 ~20행을 HTML 테이블로 가볍게 렌더 → html2canvas + 다크 톤 프레임. 또는 단순히 **Excel 아이콘 + 파일명 + "시트 N개 · 행 N개" 메타 배지**만 표시. | MVP에서는 후자(아이콘 + 메타) 권장. |
| HWPX | 브라우저에서 직접 렌더 불가. 옵션: ① 한컴 오피스 연동(불가능) ② HWPX → PDF 변환 후 PDF 썸네일 생성 — 변환기로 `@ssabrojs/hwpxjs`의 텍스트 추출만 가능하고 레이아웃 PDF 변환은 **불가**. | MVP에서는 **HWPX 아이콘 + 첫 페이지 텍스트 발췌**(hwpxcore로 추출)만 표시. |

**결론**:
- PDF만 실제 썸네일 렌더 (pdfjs-dist)
- Excel/HWPX는 **아이콘 + 메타(파일명, 페이지/시트 수, 추출 텍스트 첫 80자)**로 대체 → 성능·구현 단순화. 장기 개선 항목은 §14 리스크에 별도 기재.

import { HwpxDocument } from '@ubermensch1218/hwpxcore';

export async function fillHwpx(
  bytes: Uint8Array,
  fields: readonly DetectedMergeField[],
  ctx: MergeContext,
): Promise<Uint8Array> {
  const doc = await HwpxDocument.open(bytes);

  // 1. 모든 paragraph 순회 (본문 + 테이블 셀)
  for (const p of doc.paragraphs) {
    await replaceInParagraph(p, fields, ctx);
  }
  for (const t of doc.tables) {
    for (let r = 0; r < t.rowCount; r++) {
      for (let c = 0; c < t.colCount; c++) {
        const cellText = t.getCellText(r, c);
        const replaced = applyPlaceholders(cellText, fields, ctx);
        if (replaced !== cellText) t.setCellText(r, c, replaced);
      }
    }
  }

  return doc.save();
}

### 8-3. Excel 엔진 (`ExcelMergeEngine.ts`)

```ts
import ExcelJS from 'exceljs';

export async function fillExcel(bytes, fields, ctx): Promise<Uint8Array> {
  const wb = new ExcelJS.Workbook();
  await wb.xlsx.load(bytes.buffer);

  for (const ws of wb.worksheets) {
    ws.eachRow((row) => {
      row.eachCell({ includeEmpty: false }, (cell) => {
        const val = cell.value;
        // case 1: 단순 문자열
        if (typeof val === 'string' && val.includes('{{')) {
          cell.value = applyPlaceholders(val, fields, ctx);
        }
        // case 2: richText (서식 섞인 셀)
        else if (val && typeof val === 'object' && 'richText' in val) {
          const rt = val.richText as ExcelJS.RichText[];
          const joined = rt.map((r) => r.text).join('');
          if (joined.includes('{{')) {
            // 단순 전체 치환 → richText 구조는 첫 run에 합치고 나머지 비움
            const replaced = applyPlaceholders(joined, fields, ctx);
            cell.value = { richText: [{ ...rt[0], text: replaced }] };
          }
        }
        // case 3: 숫자/날짜는 무시
      });
    });
  }

  const buf = await wb.xlsx.writeBuffer();
  return new Uint8Array(buf as ArrayBuffer);
}
```

### 8-4. PDF 엔진 (`PdfMergeEngine.ts`)

선행 PDF Exporter 계획서가 채택한 pdf-lib 기반을 그대로 사용.

```ts
import { PDFDocument, PDFForm } from 'pdf-lib';

export async function fillPdf(bytes, fields, ctx): Promise<Uint8Array> {
  const pdf = await PDFDocument.load(bytes);

  // 전략 A — AcroForm 필드 채움 (PDF 편집기로 만든 fillable PDF)
  const form = pdf.getForm();
  for (const field of fields) {
    const value = resolveMergeValue(field, ctx);
    const pdfField = form.getFieldMaybe(field.placeholder.replace(/[{}]/g, ''));
    if (pdfField && 'setText' in pdfField) {
      (pdfField as PDFTextField).setText(value);
    }
  }
  form.flatten();  // 편집 불가로 고정

  // 전략 B — 텍스트 기반 치환은 PDF 구조상 불가능. {{...}} 직접 삽입 PDF는
  // 사전에 AcroForm field로 변환해야 지원 가능 → Phase 3 이후 "PDF 템플릿 디자이너" 과제.

  return new Uint8Array(await pdf.save());
}
```

**현실적 제약**: PDF는 텍스트가 좌표·글리프 단위로 저장되어 `{{학생이름}}` 같은 토큰 치환이 불가능. 쌤핀의 PDF mail-merge는 **AcroForm 필드 기반** PDF에만 동작. 기본 내장 서식은 **pdfme 디자이너로 AcroForm 폼 필드를 가진 PDF를 제작**하여 이 제약을 우회한다.

### 8-5. 플레이스홀더 감지 (`PlaceholderDetector.ts`)

```ts
export async function detectMergeFields(
  format: FormFormat,
  bytes: Uint8Array,
): Promise<readonly DetectedMergeField[]> {
  const text = await extractAllText(format, bytes);    // 포맷별 텍스트 추출
  const keys = parsePlaceholders(text);                // rules에서 import
  const unique = [...new Set(keys)];
  return unique.map(classifyPlaceholder);
}
```

`extractAllText`:
- HWPX: `hwpxcore`의 paragraph 순회로 run.text 추출
- Excel: `ws.eachRow` + `cell.value`/`cell.richText.text` concat
- PDF: `pdf-lib`의 `getForm().getFields()` — AcroForm field name만 대상

### 10-2. IStoragePort 확장

현재 `IStoragePort`는 JSON만 다룸. 바이너리 파일 저장을 위한 확장이 필요:

```ts
// src/domain/ports/IStoragePort.ts  (확장)
export interface IStoragePort {
  read<T>(key: string): Promise<T | null>;
  write<T>(key: string, value: T): Promise<void>;
  remove(key: string): Promise<void>;

  // ── 신규 ──
  readBinary(relPath: string): Promise<Uint8Array | null>;
  writeBinary(relPath: string, bytes: Uint8Array): Promise<void>;
  removeBinary(relPath: string): Promise<void>;
  listBinary(dirRelPath: string): Promise<readonly string[]>;
}
```

`ElectronStorageAdapter`는 Node `fs` 직접 접근, 브라우저 폴백(`LocalStorageAdapter`)은 IndexedDB로 바이너리 저장.

## 11. 썸네일 생성 전략

### 11-1. PDF (실제 렌더)

```ts
// src/infrastructure/forms/ThumbnailGenerator.ts
import * as pdfjsLib from 'pdfjs-dist';

export async function generatePdfThumbnail(
  bytes: Uint8Array,
  opts = { scale: 0.5, maxWidth: 400 },
): Promise<Uint8Array> {
  const pdf = await pdfjsLib.getDocument({ data: bytes }).promise;
  const page = await pdf.getPage(1);
  const viewport = page.getViewport({ scale: opts.scale });

  const canvas = new OffscreenCanvas(viewport.width, viewport.height);
  const ctx = canvas.getContext('2d')!;
  await page.render({ canvasContext: ctx as any, viewport }).promise;

  const blob = await canvas.convertToBlob({ type: 'image/png' });
  return new Uint8Array(await blob.arrayBuffer());
}
```

pdfjs-dist worker 설정은 Vite config에 `?url` import로 번들(`pdfjs-dist/build/pdf.worker.min.mjs`).

### 11-2. HWPX / Excel (텍스트 발췌 + 아이콘)

```ts
export async function extractHwpxPreview(bytes: Uint8Array): Promise<{
  textPreview: string;
  pageCount: number;
}> {
  const doc = await HwpxDocument.open(bytes);
  const firstPara = doc.paragraphs.slice(0, 10).map((p) =>
    p.runs.map((r) => r.text).join('')
  ).join('\n').trim().slice(0, 80);
  return { textPreview: firstPara, pageCount: doc.pageCount ?? 1 };
}

export async function extractExcelPreview(bytes: Uint8Array): Promise<{
  textPreview: string;
  sheetCount: number;
}> {
  const wb = new ExcelJS.Workbook();
  await wb.xlsx.load(bytes.buffer);
  const firstSheet = wb.worksheets[0];
  const firstRows: string[] = [];
  firstSheet?.eachRow((row, n) => {
    if (n > 3) return;
    const text = row.values?.slice(1, 5).map(String).join(' · ') ?? '';
    if (text) firstRows.push(text);
  });
  return {
    textPreview: firstRows.join('\n').slice(0, 80),
    sheetCount: wb.worksheets.length,
  };
}
```

카드 렌더링 시 `FormCard.tsx`가 포맷별로 분기:
- PDF: `<img src={thumbnailDataUrl}>` 실제 썸네일
- HWPX: `<div>` + 배경색 `sp-card`, 중앙에 한글 아이콘, 하단에 `textPreview`
- Excel: 동일 패턴, 초록 아이콘
