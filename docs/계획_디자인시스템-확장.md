# 디자인 시스템 확장 계획 — 쌤핀(SsamPin) v1.11.x 예정

> 2026-04-24 작성. 레퍼런스: `docs/examples/` 96개 중 선별한 Tier S 3개 + Tier A 3개.
> 작성: designer-high(Tier S) + frontend-architect(Tier A) 서브에이전트 병렬 분석.

## 0. 목적

쌤핀 현 디자인 시스템의 4대 약점을 레퍼런스 매핑으로 해소한다.

| 약점 | 현황 | 해결 레퍼런스 |
|------|------|---------------|
| Shadow 부재 | `--sp-card-shadow: none` | Notion 3-layer(라이트) + Linear 5-layer(다크) |
| Motion 토큰 부재 | 컴포넌트마다 duration/easing 제각각 | Linear `.16s cubic-bezier(0.25,0.46,0.45,0.94)` |
| Radius 단일 | `--sp-card-radius: 4px` 하나 | Linear 4/6/8/12/16 5단 |
| Weight 2단 | regular/bold만 사용 | Linear 400/510/590/680 Pretendard Variable axis |
| Sidebar 공간감 | 웹앱 느낌, 섹션 리듬 부족 | Arc 사이드바 패턴 |
| 일정 페이지 | dot 기반 이벤트 표시, 위계 약함 | Cal.com 이벤트 칩 + 뷰 토글 |
| 단축키 UX 부재 | Electron 앱인데 단축키 시각화 없음 | Raycast kbd + Ctrl+K 팔레트 |

## A. 토큰 확장 (Tier S — designer-high 제안)

### A-1. 레퍼런스 → 채택 매핑 요약

**Linear에서 가져올 것**
- Weight 4단 `400 / 510 / 590 / 680` (Pretendard Variable axis 활용)
- Duration 2단 `120ms / 160ms` + Ease `cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- 5-layer dual shadow (다크 전용)
- Radius 배수 4 / 6 / 8 / 12 / 16 (24/32 제외 — 데스크톱 밀도형에 과함)

**Notion에서 가져올 것**
- 3-level multi-layer shadow (라이트 전용, L100/L200/L300)
- 링크 hover는 color가 아니라 `text-decoration-color` transition
- 기본 카드 radius 12px

**Toss에서 가져올 것**
- 버튼 `:active { scale: 0.98 }` + `:hover { translateY(-1px) }`
- CTA blue-tinted shadow `0 8px 24px rgba(35, 131, 226, 0.25)`
- (Toss의 20px/28px radius는 **거절** — 쌤핀 데스크톱 밀도 정체성 훼손)

```js
      medium:   'var(--sp-weight-medium)',     // 510

      bold:     'var(--sp-weight-bold)',       // 680
    },

    borderRadius: {
      'sp-xs':   'var(--sp-radius-xs)',
      'sp-sm':   'var(--sp-radius-sm)',
      'sp-md':   'var(--sp-radius-md)',
      'sp-lg':   'var(--sp-radius-lg)',
      'sp-xl':   'var(--sp-radius-xl)',
      'sp-pill': 'var(--sp-radius-pill)',
    },

    boxShadow: {
      'sp-none':   'var(--sp-shadow-none)',
      'sp-sm':     'var(--sp-shadow-sm)',
      'sp-md':     'var(--sp-shadow-md)',
      'sp-lg':     'var(--sp-shadow-lg)',
      'sp-accent': 'var(--sp-shadow-accent)',
    },

    transitionDuration: {
      'sp-quick': 'var(--sp-duration-quick)',
      'sp-base':  'var(--sp-duration-base)',
      'sp-slow':  'var(--sp-duration-slow)',
    },

    transitionTimingFunction: {
      'sp-out':       'var(--sp-ease-out)',
      'sp-out-cubic': 'var(--sp-ease-out-cubic)',
      'sp-in-out':    'var(--sp-ease-in-out)',
    },
  },
}
```

> `sp-` 접두어 필수 이유: Tailwind 기본 `shadow-sm`/`rounded-lg`와 충돌 방지. 기존 컴포넌트 무중단 유지 + 신규 컴포넌트만 `shadow-sp-md` 등으로 opt-in.

## B. 컴포넌트 개선 (Tier A — frontend-architect 제안)

### B-1. Sidebar 리팩터 (Arc 레퍼런스)

**현재 파일**: `src/adapters/components/Layout/Sidebar.tsx`

**Arc 기준 수치**: 항목 padding `12px 16px`, section-gap `24px`, radius `12px`, 아이콘-레이블 gap `8px`, expanded `240px`, collapsed `64px`.

**변경 포인트**:
- 로고 영역 `p-4` → `px-4 pt-5 pb-3`
- 네비게이션 래퍼 `px-2 py-2 gap-1` → `px-3 py-3 gap-0.5`
- 아이템 버튼 `py-3 px-4 rounded-xl` → `py-2.5 px-3 rounded-lg`
- 아이템 내 `gap-3` → `gap-2.5`
- 비활성 hover `hover:bg-sp-text/5` → `hover:bg-white/[0.06]` (다크) / `hover:bg-black/[0.05]` (라이트)
- **섹션 구분선 추가**: 네비를 '대시보드/시간표/일정' + '메모/노트/할일' 2그룹으로 분리. `<div className="h-px bg-sp-border/40 mx-3 my-1.5" role="separator" />`
- Expanded width `w-64 (256px)` → `w-56 (224px)` (Arc 240px 근사값)
- 전환 `duration-200 ease-in-out` → `duration-150 ease-out` (데스크톱은 200ms가 느림)

**축소/확장 분리**:
- Collapsed 모드에서 avatar(`w-8 h-8`)만 하단에 남기고 SyncStatusBar/DriveSyncIndicator는 `title` tooltip으로 대체

### B-2. 일정 페이지 개선 (Cal 레퍼런스)

**현재 파일**: `src/adapters/components/Schedule/CalendarView.tsx`, `Schedule.tsx`, `EventList.tsx`

**이벤트 칩 개선**:
- 현재 dot 방식 → 제목 보이는 칩 방식 전환, max 2개 + `+N` 오버플로 뱃지
- 칩 클래스: `inline-flex items-center px-1.5 py-0.5 rounded text-[11px] font-medium truncate max-w-full`
- 배경 `bg-{color}/20` (라이트는 `/30`), 텍스트 `text-{color}`
- day cell에 `overflow-hidden` 필수 (multiday bar 충돌 방지)

**Day Cell 상태**:
- 오늘: `ring-2 ring-sp-accent ring-offset-1 ring-offset-sp-bg`
- 선택됨: `bg-sp-accent/15 font-semibold`
- Hover: `hover:bg-white/[0.05] transition-colors duration-150`
- 공휴일: 숫자 `text-red-400` + 이름 `text-[10px] text-red-400/70`
- 토요일: `text-blue-400/80`, 일요일: `text-red-400/80`

**뷰 토글(월간/학기/연간)을 Cal.com 탭 패턴으로**:
```
래퍼: flex items-center bg-sp-surface/60 rounded-lg p-0.5 border border-sp-border
개별 탭: px-3 py-1.5 rounded-md text-sm font-medium transition-all duration-150
활성: bg-sp-card text-sp-text shadow-sm
비활성: text-sp-muted hover:text-sp-text
```

**EventList 개선**:
- 이벤트 항목 `border border-sp-border rounded-lg p-3 mb-2 hover:border-sp-accent/30 hover:bg-sp-card/50`
- 좌측 컬러 바 `w-1 h-full rounded-full` (카테고리 색상) — Cal.com left-border accent
- 날짜 그룹 헤더 `text-xs font-semibold text-sp-muted uppercase tracking-wider mb-2 mt-4`

**Kbd 스펙**:

**CommandPalette 스펙**:
- Overlay: `fixed inset-0 bg-black/50 backdrop-blur-sm z-50`

## D. 리스크 & 주의사항

### D-1. 시각 회귀

- `--sp-card-radius` **기본값 4px 유지**. 신규 컴포넌트만 `rounded-sp-lg (12px)` opt-in 사용. 한 번에 전역 승격 금지.
- 출석부 셀처럼 밀도형 뷰는 `rounded-sp-xs (4px)` 명시.

### D-2. 라이트/다크 shadow 호환

- Notion 3-layer(옅은 검정)는 다크에서 안 보임 → 다크 블록에서 Linear 5-layer로 **반드시 오버라이드**.
- Linear 5-layer는 라이트에서 너무 진함 → 라이트 블록에서 Notion 3-layer로 오버라이드.

### D-3. Pretendard Variable axis

- `font-weight: 510/590/680`은 Pretendard Variable woff2가 로드되어야 작동.
- Static Pretendard만 있으면 500/600/700으로 반올림 → 기능은 유지되지만 Linear만큼 미세 계단은 나오지 않음.
- **Phase 1 시작 전 확인**: `src/assets/fonts/` 또는 CDN에서 `PretendardVariable.woff2` 로드 여부.

### D-4. Electron 성능

- 4-layer 이상 shadow가 한 화면에 10개+ 렌더되면 60fps 유지 어려울 수 있음.
- **규칙**: 리스트 행(학생 30명 등)에는 shadow 금지. hover 시만 shadow 적용.
- `transition: box-shadow`는 무거움 → `transition: transform, opacity`만 transition, shadow는 즉시 교체.

### D-5. CommandPalette 글로벌 단축키

- Electron `globalShortcut` 사용 금지 (교사 수업 중 간섭 위험).
- `window.addEventListener('keydown')` renderer 처리.
- 텍스트 입력 포커스 시 가드 필수.
- `role="dialog" aria-modal="true"` + focus trap + 결과 `role="listbox"` / `role="option"` 접근성 확보.

### D-6. 네임스페이스 규율

- 이번 확장으로 `--sp-shadow-*`, `--sp-radius-*`, `--sp-weight-*`, `--sp-duration-*`, `--sp-ease-*` 5개 하위 네임스페이스 신설.
- `CLAUDE.md`의 "디자인 시스템" 섹션에 이 5개 네임스페이스 명시.
- 이후 모든 추가 토큰은 `--sp-{namespace}-{variant}` 규칙 고정.
