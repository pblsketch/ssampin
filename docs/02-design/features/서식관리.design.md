## 2. 도메인 계약 (Contracts)

### 2-1. `IStoragePort` 확장

**현재**:
```ts
export interface IStoragePort {
  read<T>(filename: string): Promise<T | null>;
  write<T>(filename: string, data: T): Promise<void>;
}
```

**확장 후** (`src/domain/ports/IStoragePort.ts`):
```ts
export interface IStoragePort {
  // 기존 (JSON)
  read<T>(filename: string): Promise<T | null>;
  write<T>(filename: string, data: T): Promise<void>;

  // 신규 (바이너리) — Phase 1
  readBinary(relPath: string): Promise<Uint8Array | null>;
  writeBinary(relPath: string, bytes: Uint8Array): Promise<void>;
  removeBinary(relPath: string): Promise<void>;
  listBinary(dirRelPath: string): Promise<readonly string[]>;
}
```

**호환성**: 기존 `read/write`는 의미 변경 없음. 바이너리 메서드는 `userData/` 기준 상대 경로를 받는다. 브라우저 폴백(`LocalStorageAdapter`)은 IndexedDB로 바이너리 저장.

**계약 조건**:
- `writeBinary`: 중간 디렉토리 자동 생성. 동일 경로 덮어쓰기 허용.
- `readBinary`: 미존재 시 `null`. 예외 던지지 않음.
- `removeBinary`: 미존재 시 no-op.
- `listBinary`: 반환값은 **파일명만** (디렉토리 제외, 상대경로 아님).

### 2-2. `IFormTemplateRepository`

`src/domain/repositories/IFormTemplateRepository.ts`:
```ts
export interface IFormTemplateRepository {
  list(): Promise<readonly FormTemplate[]>;
  get(id: string): Promise<FormTemplate | null>;
  create(template: FormTemplate, fileBytes: Uint8Array, thumbnail?: Uint8Array): Promise<void>;
  update(id: string, patch: Partial<Omit<FormTemplate, 'id' | 'isBuiltin' | 'createdAt'>>): Promise<void>;
  delete(id: string): Promise<void>;
  readFile(id: string): Promise<Uint8Array>;
  readThumbnail(id: string): Promise<Uint8Array | null>;
  listCategories(): Promise<readonly FormCategory[]>;
  upsertCategory(c: FormCategory): Promise<void>;
  deleteCategory(id: string): Promise<void>;
}
```

**에러 정책**:
- `readFile`에서 파일 누락 시 `FormFileMissingError(id)` throw (복구 가능 에러).
- `delete(id)`가 `isBuiltin=true`에 호출되면 `BuiltinFormProtectedError` throw.
- `deleteCategory`에서 해당 카테고리를 쓰는 서식이 있으면 `CategoryInUseError(count)` throw — UI에서 이관 대화 제공.

### 2-3. 엔티티 불변성

- `FormTemplate`, `FormCategory` 모두 `readonly` 필드로 freeze.
- 수정은 repository의 `update()` 경유 — store 레이어에서 직접 객체 변형 금지.

## 3. Electron IPC 계약

**채널명 규칙**: `forms:{action}`, 전부 `ipcMain.handle` (양방향).

| 채널 | 페이로드 | 반환 | 비고 |
|---|---|---|---|
| `forms:writeBinary` | `{ relPath: string; bytes: ArrayBuffer }` | `void` | `Uint8Array`는 IPC 직렬화 비용 이슈로 `ArrayBuffer`로 교환 |
| `forms:readBinary` | `{ relPath: string }` | `ArrayBuffer \| null` | |
| `forms:removeBinary` | `{ relPath: string }` | `void` | |
| `forms:listBinary` | `{ dirRelPath: string }` | `string[]` | 파일명만 |

**보안 가드** (`electron/main.ts`):
- 모든 `relPath`를 `path.resolve(userData, relPath)`로 정규화 후 `userData` 하위인지 검증. 경계 이탈 시 에러 throw.
- 허용 확장자 화이트리스트: `.hwpx`, `.pdf`, `.xlsx`, `.png` (썸네일). 그 외는 거부.

**preload 노출** (`electron/preload.ts`):
```ts
contextBridge.exposeInMainWorld('electronAPI', {
  // 기존...
  forms: {
    writeBinary: (relPath: string, bytes: ArrayBuffer) => ipcRenderer.invoke('forms:writeBinary', { relPath, bytes }),
    readBinary:  (relPath: string) => ipcRenderer.invoke('forms:readBinary', { relPath }),
    removeBinary:(relPath: string) => ipcRenderer.invoke('forms:removeBinary', { relPath }),
    listBinary:  (dirRelPath: string) => ipcRenderer.invoke('forms:listBinary', { dirRelPath }),
  },
});
```

## 4. Usecase 시그니처 (Phase 1)

`src/usecases/forms/`:

```ts
// ListForms.ts — 순수 위임 + domain rules
export function listForms(
  all: readonly FormTemplate[],
  filter: FormFilter,
  sort: FormSort,
): readonly FormTemplate[];

// CreateForm.ts
export interface CreateFormInput {
  name: string;
  format: FormFormat;
  categoryId: string;
  tags: readonly string[];
  fileBytes: Uint8Array;  // 드롭존에서 읽은 원본
}
export async function createForm(
  input: CreateFormInput,
  deps: {
    repo: IFormTemplateRepository;
    thumbnailer: IThumbnailer;
    previewExtractor: IPreviewExtractor;
    idGen: () => string;  // uuid
    now: () => Date;
  },
): Promise<FormTemplate>;

// UpdateForm.ts — 이름/카테고리/태그/즐겨찾기
export async function updateForm(
  id: string,
  patch: { name?: string; categoryId?: string; tags?: readonly string[]; starred?: boolean },
  repo: IFormTemplateRepository,
): Promise<void>;

// DeleteForm.ts — 내장 가드 포함
export async function deleteForm(id: string, repo: IFormTemplateRepository): Promise<void>;

// PrintForm.ts — Electron printToPDF 경유 OS 프린트

  id: string,
  repo: IFormTemplateRepository,

```

## 5. 어댑터 계층 설계

### 5-1. `useFormStore` (Zustand) shape

```ts
interface FormStoreState {
  // 데이터
  forms: readonly FormTemplate[];
  categories: readonly FormCategory[];
  loaded: boolean;

  // UI 상태
  filter: FormFilter;          // { format?, categoryId?, starred?, query? }
  sort: FormSort;              // 'recent' | 'name' | 'created'
  selectedId: string | null;   // 미리보기 모달 대상
  uploadModalOpen: boolean;

  // 액션 (모두 async; usecase 경유)
  loadAll(): Promise<void>;
  create(input: CreateFormInput): Promise<void>;
  update(id: string, patch: UpdateFormPatch): Promise<void>;
  remove(id: string): Promise<void>;
  toggleStar(id: string): Promise<void>;
  setFilter(f: Partial<FormFilter>): void;
  setSort(s: FormSort): void;
  select(id: string | null): void;
  openUpload(): void;
  closeUpload(): void;
}
```

**파생 상태는 selector로** (`useFormStore(selectFilteredForms)`) — store 안에 캐시하지 않는다. `filterForms/sortForms`는 domain rules 순수함수라 재렌더 비용 낮음.

## 6. 데이터 흐름 시퀀스

### 6-1. 업로드 (CreateForm)

```
FormUploadModal
  └─ [user drops file.hwpx]
  └─ FileReader → Uint8Array
  └─ useFormStore.create(input)
        └─ createForm(input, deps)
              ├─ idGen() → uuid
              ├─ previewExtractor.extract(format, bytes)  // textPreview
              ├─ (format=pdf) thumbnailer.generate(bytes) // PNG bytes
              ├─ template = { ...input, id, textPreview, ... }
              └─ repo.create(template, bytes, thumbnail?)
                    ├─ storage.writeBinary(`forms/${id}.${ext}`, bytes)
                    ├─ storage.writeBinary(`forms/thumbs/${id}.png`, thumb)  // pdf만
                    └─ storage.write('forms', [...existing, template])
```

### 6-2. 목록 렌더 (초기)

```
FormsPage mount
  └─ useFormStore.loadAll()
        ├─ repo.list()           // forms.json 읽기
        ├─ repo.listCategories() // builtin + user
        └─ setState({ forms, categories, loaded: true })

FormGrid
  └─ const visible = useFormStore(s => filterForms(sortForms(s.forms, s.sort), s.filter))
```

### 6-3. 썸네일 지연 로드

- PDF 카드 첫 마운트 시 `repo.readThumbnail(id)`를 `useEffect`로 가져와서 `URL.createObjectURL(blob)` 부여.
- 언마운트 시 `revokeObjectURL`로 누수 방지.

## 8. pdfjs-dist 번들 전략

- 설치: `npm i pdfjs-dist`
- Vite 설정(`vite.config.ts`): worker는 별도 청크 — `pdfjs-dist/build/pdf.worker.min.mjs`를 `?url` import.
- `PdfJsThumbnailer` 생성자에서 `GlobalWorkerOptions.workerSrc = workerUrl` 설정.
- **코드 스플릿**: `FormsPage`는 `React.lazy`로 지연 로드 → 다른 페이지는 pdfjs 번들 영향 없음.
- Electron renderer의 `OffscreenCanvas` 지원 확인 (Electron 33 = Chromium 130, 지원).

## 9. 테스트 계획 (Phase 1)

### 9-1. 단위 (`test/domain/rules/formTemplateRules.test.ts`)
- `parsePlaceholders`: 기본/공백허용/중첩무시/다수추출 — 4 케이스
- `filterForms`: format·category·starred·query 조합 6 케이스
- `sortForms`: recent/name/created 3 케이스

### 9-2. 통합 (repository 레벨, 수동 또는 단위)
- `JsonFormTemplateRepository.create` → `list` → `readFile` round-trip
- `delete(builtinId)` → `BuiltinFormProtectedError`

### 9-3. 수동 (릴리즈 전)
- Windows: 드롭존에 HWPX/PDF/Excel 각 1개 업로드 → userData/forms/ 실제 파일 생성 확인 (탐색기로)
- 앱 재시작 → 목록 복구
- PDF 썸네일이 실제 첫 페이지인지 확인
- 삭제 후 userData에서 실물 파일·썸네일·메타 모두 사라졌는지

> **런타임 검증 원칙** (MEMORY): "동작한다"의 기준은 UI 토스트가 아니라 **userData 실제 파일 바이트**.

## 10. 리스크·완화 (Phase 1 국한)

| 리스크 | 완화 |
|---|---|
| `ArrayBuffer` IPC 전송 비용 (수 MB 파일) | Electron 32+는 `SharedArrayBuffer` 아닌 일반 transfer도 빠름. 10MB 이상 제한을 UI에서 걸어 초기 이슈 차단. |
| pdfjs-dist worker 경로 오류 (Electron file:// 컨텍스트) | `?url` import로 Vite가 절대경로 주입. 개발/빌드 양쪽에서 테스트. |
| IndexedDB 브라우저 폴백 미구현 영역 | 브라우저 모드는 Phase 1에서 "업로드만 가능, 재시작 시 유지 안됨" 경고 토스트로 우선 처리 → Phase 2에서 IndexedDB 완성. |
| 카테고리 삭제 시 고아 서식 | `CategoryInUseError` 후 UI에서 "다른 카테고리로 이동" 대화. |
| 파일 경로 traversal (IPC) | main 프로세스에서 `path.resolve` 정규화 + userData 경계 검증. |

- Phase 2(내장 10종)는 `scripts/build-builtin-forms.ts`에서 이 설계의 repo 계약을 그대로 사용. 앱 부팅 시 `ImportBuiltins` usecase가 `repo.create(builtinTemplate, bytes)` 반복 호출.

- `MergeField.ts` 타입은 Phase 1에서 선언만 해두고(빈 배열로 저장), Phase 3에서 `PlaceholderDetector`가 채운다.
