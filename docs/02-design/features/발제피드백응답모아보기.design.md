## 1. Architecture Overview

### 1.1 Design Goals

- 사용자가 실시간 설문 종료 직후 **한 화면**에서 요약·테이블·개별 응답을 모두 조망
- 과거 결과(`useToolResultStore`) 열람 시에도 **동일 컴포넌트 재사용** — 두 진입점(run 직후, 과거) UI 병존 제거
- Excel 다운로드는 **셀 직렬화 규칙**이 명확해야 필터/정렬이 동작 (멀티초이스 배열 처리가 핵심)
- 응답자 익명/구분 모드 토글 — 민감한 교실 피드백 시나리오에서 기본 보호
- 향후 이미지 응답(Phase 3, A안) 추가 시 **타입만 확장**하고 UI 구조는 그대로 — 썸네일 셀이 기존 테이블에 삽입되는 형태

### 1.2 Design Principles

- **단일 결과 뷰어**: SpreadsheetView가 `results` 단계 + 과거 결과 모달 모두 담당 (중복 UI 제거)
- **계층 분리**: 집계/통계/직렬화 순수 함수는 `domain/rules/`, 렌더링은 `adapters/components/`
- **점진적 확장**: 질문 타입 추가 시(`image` 등) `value` 유니온만 확장, 렌더러는 switch 분기 추가
- **오프라인 무영향**: 전 컴포넌트가 로컬 `useToolResultStore` 데이터로만 동작 — 네트워크 의존 0

### 2.3 Excel 직렬화 규칙 (domain/rules)

```typescript
// src/domain/rules/toolResultSerialization.ts

/**
 * 테이블 뷰 · Excel Sheet 2 공통 셀 값 변환 규칙.
 * 순수 함수 — React/ExcelJS 의존 없음.
 *
 * 타입 참고:
 * - MultiSurveyTemplateQuestion.options는 non-optional readonly 배열 (ToolTemplate.ts:27)
 * - text/scale 질문에서는 빈 배열이 올 수 있음 (유효한 상태)
 */
export function serializeAnswerCell(
  question: MultiSurveyTemplateQuestion,
  answer: { value: string | string[] | number } | undefined,
): { display: string; raw: string | number | null } {
  if (!answer) return { display: '', raw: null };

  switch (question.type) {
    case 'single-choice': {
      const opt = question.options.find(o => o.id === answer.value);
      return { display: opt?.text ?? '', raw: opt?.text ?? '' };
    }
    case 'multi-choice': {
      const ids = Array.isArray(answer.value) ? answer.value : [];
      const texts = ids
        .map(id => question.options.find(o => o.id === id)?.text)
        .filter((t): t is string => !!t);
      return { display: texts.join('; '), raw: texts.join('; ') };
    }
    case 'text': {
      const text = typeof answer.value === 'string' ? answer.value : '';
      return { display: text, raw: text };
    }
    case 'scale': {
      const n = typeof answer.value === 'number' ? answer.value : null;
      return { display: n !== null ? String(n) : '', raw: n };
    }
  }
}

/**
 * 제출 순서 기반 응답자 라벨.
 * Phase 1은 로그인 기능 없음 → 파라미터 단순화.
 * Phase 2+ 로그인 도입 시 `formatRespondentLabel(index, { anonymous, submittedBy })`로 확장.
 */
export function formatSubmissionLabel(submissionIndex: number): string {
  return `응답 ${submissionIndex + 1}`;
}
```

## 3. Component Specifications

### 3.1 SpreadsheetView

**Props (두 경로 모두 같은 컴포넌트 재사용을 위한 입력 변형):**
```typescript
interface SpreadsheetViewProps {
  // inline: ToolMultiSurvey results 단계 — 원시 데이터 직접 주입 (아직 store 저장 전일 수 있음)
  // modal: PastResultsView — 이미 store에 저장된 ToolResult 전체
  source:
    | { mode: 'inline'; data: MultiSurveyResultData }
    | { mode: 'modal'; result: ToolResult };  // result.data.type === 'multi-survey'
  onClose?: () => void;        // modal 전용
  inlineFooter?: React.ReactNode;  // inline 전용 — 하단 액션 슬롯 (저장/새 설문 등)
}

// 내부에서 항상 MultiSurveyResultData로 정규화 후 사용 (타입 가드 포함)
const data: MultiSurveyResultData | null = useMemo(() => {
  if (source.mode === 'inline') return source.data;
  if (source.result.data.type === 'multi-survey') return source.result.data;
  return null;  // 방어적: multi-survey가 아닌 ToolResult가 들어오면 fallback 렌더
}, [source]);
```

**데이터 조회 경로 (modal 모드):**
- `PastResultsView`는 `useToolResultStore`의 `results[]`에서 카드 클릭 시 해당 `ToolResult` 객체를 그대로 `<SpreadsheetView source={{ mode: 'modal', result }} />`로 전달
- 별도 fetch/API 없음 — 완전 로컬 상태

**🚨 Modal 모드 필수 구현: `createPortal(document.body)`**
- **원인:** 상위 컨테이너(예: `ToolLayout`)에 `transform` / `filter` / `will-change: transform` 등이 걸려 있으면 CSS 사양상 `position: fixed`의 containing block이 viewport가 아닌 해당 조상으로 전환됨. 결과적으로 모달이 viewport 밖으로 잘리거나 오프셋됨.
- **해결:** 모달 JSX를 `createPortal(<div className="fixed inset-0 z-[100] ...">...</div>, document.body)`로 감싸 DOM 트리에서 body 직속으로 이동시켜 조상 영향 차단.
- **z-index:** `z-[100]`으로 ToolLayout 내부 요소(`z-50` 이하)보다 위에 위치.
- **배경:** 모달 내부 컨테이너에 `bg-sp-bg` 필수 (투명 방지).

```typescript
import { createPortal } from 'react-dom';

if (source.mode === 'modal') {
  return createPortal(
    <div className="fixed inset-0 z-[100] flex items-center justify-center bg-black/60 p-4"
         onClick={(e) => { if (e.target === e.currentTarget) onClose?.(); }}>
      <div className="h-[90vh] w-full max-w-6xl overflow-hidden rounded-xl border border-sp-border bg-sp-bg shadow-2xl">
        {body}
      </div>
    </div>,
    document.body,
  );
}
```

**State:**

| state | 기본값 | 탭 전환 시 | 비고 |
|-------|-------|-----------|------|
| `activeTab: 'summary' \| 'table' \| 'individual'` | `'summary'` | — | — |
| `anonymous: boolean` | `true` | **유지** | Plan의 "응답자 구분 표시 OFF" = Design의 `anonymous=true`. 의미는 동일, 네이밍만 반전된 매핑. 문서상 **Design 네이밍(`anonymous`) 기준**으로 통일 |
| `filterOption: { questionId; optionId } \| null` | `null` | **table → 타탭 이탈 시 리셋** | table 전용 — TableTab 내부 state로 구현 (unmount로 자연 리셋) |

### 3.2 SummaryTab

- 질문별 블록을 수직으로 렌더
- `aggregateAll(data)` 호출 → 반환된 `QuestionAggregate[]`를 타입별 컴포넌트로 분기 렌더:
  - `ChoiceAggregate` → 막대 그래프 (`PastResultsView.tsx`의 `PollDetail` 스타일 재사용)
  - `ScaleAggregate` → 평균/중앙값 + 분포 바
  - `TextAggregate` → 응답 수 + 펼침 리스트 (최대 5개 + "모두 보기" 버튼)
- 상단에 "총 참여 N명 · 완료율 M%"

### 3.3 TableTab

**CSS Grid 구조:**
- `grid-template-columns: auto repeat(N, minmax(140px, 1fr))` (첫 열: 응답자 라벨, 나머지: 질문 N개)
- 헤더 행: `sticky top-0 bg-sp-surface` + 열 클릭으로 정렬 방향 토글
- 본문 행: `sp-card` 배경, `sp-border` divider

**셀 렌더링:**
- `serializeAnswerCell(question, answer)` 호출 → `display` 문자열을 셀에 표시
- 텍스트 길이 > 40자: `truncate` + hover 시 tooltip으로 전문 표시
- 스케일: 숫자 + "★" 5개 중 채움 아이콘 (선택)

**정렬/필터:**
- 정렬은 `display` 문자열 기준 (스케일은 숫자 비교로 분기)
- 필터: `filterOption` state가 설정되면 해당 옵션 선택 응답만 남김
- 검색: `searchKeyword`가 text 질문의 답에 포함된 응답만 남김

**빈 셀:** 미응답 시 `bg-sp-surface/40`로 회색 처리 + "—"

### 3.4 IndividualTab

- `submissions` 배열을 순회, 한 번에 1명만 표시
- 좌우 화살표 버튼 (키보드 `←/→` 단축키 지원)
- 카드 구조: 응답자 라벨 + 제출 시각 + 질문/답변 쌍 수직 배열
- 답변은 질문 타입별로 다르게 렌더 (스케일은 별 + 숫자, 멀티초이스는 칩)

### 3.5 익명 토글

- `<Switch />` 스타일 (프로젝트 기존 Switch 컴포넌트 재사용 — 없으면 인라인 구현)
- `anonymous: true`일 때 모든 라벨은 `응답 1, 응답 2, ...` (제출 순서)
- `anonymous: false`일 때도 Phase 1에서는 동일 (로그인 기능 없음) — Phase 2+에서 이름 노출 분기 추가

## 4. Excel Export Specification

### 4.1 함수 시그니처

[src/infrastructure/export/ExcelExporter.ts](../../../src/infrastructure/export/ExcelExporter.ts):

```typescript
/**
 * 반환 타입은 기존 exporter 함수들과 통일하여 `Promise<ArrayBuffer>`.
 * ExcelJS 최신 버전의 `workbook.xlsx.writeBuffer()`는 Node Buffer를 반환하므로 `as ArrayBuffer` 캐스트 사용.
 * `Blob`으로 감쌀 때 양쪽 타입 모두 정상 동작.
 */
export async function exportToolResultToExcel(
  result: ToolResult,
  options?: { anonymous?: boolean },
): Promise<ArrayBuffer> {
  const workbook = new ExcelJS.Workbook();
  switch (result.data.type) {
    case 'multi-survey':
      addMultiSurveySheets(workbook, result.data, options?.anonymous ?? true);
      break;
    case 'poll':
      addPollSheet(workbook, result.data);
      break;
    case 'survey':
      addSurveySheet(workbook, result.data);
      break;
    case 'wordcloud':
      addWordCloudSheet(workbook, result.data);
      break;
    // 토론 타입은 exhaustive 검사 충족을 위해 명시적 throw (ToolResultData union에 이미 포함)
    case 'valueline-discussion':
    case 'trafficlight-discussion':
      throw new Error(`이 도구 타입은 엑셀 내보내기를 아직 지원하지 않습니다: ${result.data.type}`);
  }
  return (await workbook.xlsx.writeBuffer()) as ArrayBuffer;
}

/**
 * PastResultsView가 아닌 경로(`ToolMultiSurvey.results` 단계)에서 편의상 사용.
 * `ToolResult` 껍데기를 만들지 않아도 되도록 오버로드 제공.
 */
export async function exportMultiSurveyDataToExcel(
  data: MultiSurveyResultData,
  options?: { anonymous?: boolean },
): Promise<ArrayBuffer> {
  const workbook = new ExcelJS.Workbook();
  addMultiSurveySheets(workbook, data, options?.anonymous ?? true);
  return (await workbook.xlsx.writeBuffer()) as ArrayBuffer;
}
```

### 4.2 multi-survey 시트 구성

**Sheet 1 "요약":**
| A (질문 번호) | B (질문) | C (유형) | D~ (유형별 통계) |
|---|---|---|---|
| 1 | ... | 단일선택 | 옵션1: N명 (X%) · 옵션2: ... |
| 2 | ... | 복수선택 | (동일) |
| 3 | ... | 텍스트 | 응답 수: N |
| 4 | ... | 척도 | 평균 X.X · 중앙값 X |

**Sheet 2 "전체 응답":**
- **A열: 응답자 라벨** (`formatSubmissionLabel(idx)`)
- **B열: 제출 시각** (`YYYY-MM-DD HH:mm` 포맷 — Excel 정렬/필터 용이)
- **C열~**: 질문별 답 (`serializeAnswerCell(q, answer).raw`)
- 셀 직렬화는 §2.3 규칙 준수
- 미응답 셀은 `null` 값 + `sp-surface` 회색 배경(`FFF3F4F6`)으로 구분

**Sheet 3 "워드 빈도" (텍스트 질문이 1개 이상일 때만):**
- 모든 text 질문의 응답을 합쳐 간이 토크나이저(공백/구두점 분리) → 단어 빈도 집계
- `applyHeaderStyle`/`applyCellStyle`로 기존 ssampin 스타일 일관성 유지

### 4.3 기타 타입 시트 (간이 구성)

- `poll` → 단일 시트: 질문 + 옵션별 투표수/비율 + 총 투표수
- `survey` → 단일 시트: 질문 + 응답 목록 (제출 순서)
- `wordcloud` → 단일 시트: 단어 + 빈도 (내림차순)

### 4.4 다운로드 트리거 (adapters)

```typescript
// SpreadsheetView 내부
import { format } from 'date-fns';  // 프로젝트 기존 date-fns ^3.6 사용 (CLAUDE.md 기술 스택)
import { exportMultiSurveyDataToExcel } from '@infrastructure/export/ExcelExporter';

/** 파일명 안전 문자열 (경로 주입 방지 — §7 보안 요건) */
function sanitizeFileName(name: string): string {
  return name.replace(/[\\/:*?"<>|]/g, '_').trim() || '결과';
}

async function handleExport() {
  try {
    const buffer = await exportMultiSurveyDataToExcel(data, { anonymous });
    const blob = new Blob([buffer], {
      type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet'
    });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `${sanitizeFileName(data.title)}_응답모음_${format(new Date(), 'yyyyMMdd')}.xlsx`;
    a.click();
    URL.revokeObjectURL(url);
  } catch (e) {
    console.error('Excel export failed', e);
    // TODO: Toast UI가 프로젝트에 있다면 연결, 없으면 alert fallback
  }
}
```

`PastResultsView` 모달 경로에서는 이미 `ToolResult` 객체를 보유하므로 `exportToolResultToExcel(result, { anonymous })` 호출. 껍데기 생성 불필요.

## 7. Security Considerations

- 로컬 Electron/브라우저 동작 — 외부 전송 0건 → CSRF/XSS 표면 없음
- 단, **사용자 입력 텍스트를 DOM에 렌더** 시 XSS 주의: React 기본 escape로 충분. `dangerouslySetInnerHTML` 사용 금지.
- Excel 다운로드 파일명에 사용자 입력(`data.title`) 포함 → 파일명 안전 문자로 sanitize (`/[\\/:*?"<>|]/g` 제거)
- 응답자 식별이 로그로 남지 않도록 console 출력 금지
- [ ] 입력 검증 (XSS 방지: React escape 신뢰, `dangerouslySetInnerHTML` 미사용)
- [ ] 파일명 sanitize (경로 주입 방지)
- [ ] 응답 텍스트 로그 미출력
