## 1. 기능 개요

### 1.1 한 줄 요약

쌤핀의 일정을 구글 캘린더와 양방향으로 동기화하여, 한쪽에서 일정을 추가/수정하면 다른 쪽에도 자동 반영되는 기능.

### 1.2 왜 필요한가?

| 현재 상황 | 구글 캘린더 연동 후 |
|----------|-------------------|
| 쌤핀에 일정 입력 + 폰 구글 캘린더에도 별도 입력 | 한 곳에만 입력하면 양쪽 다 반영 |
| 회의 중 폰에서 구글 캘린더로 일정 확인 → 쌤핀 일정 안 보임 | 구글 캘린더에서 쌤핀 일정까지 다 보임 |
| 학교 밖에서 일정 확인 불가 | 구글 캘린더 앱으로 어디서든 확인 |
| 알림은 쌤핀 실행 시에만 | 구글 캘린더 푸시 알림으로 언제든 |

### 1.3 핵심 원칙

1. **UX 최우선**: 클릭 3번이면 연동 완료
2. **양방향**: 쌤핀 → 구글, 구글 → 쌤핀 모두 자동 동기화
3. **선택적 동기화**: 카테고리별로 동기화 여부 선택 가능
4. **오프라인 안전**: 오프라인에서도 일정 관리 가능, 온라인 복귀 시 자동 동기화
5. **출처 구분**: 구글에서 온 일정과 쌤핀 고유 일정을 뱃지로 구분

### 1.4 프로덕션 전환 계획

#### 왜 프로덕션 전환이 필요한가?

쌤핀은 이미 **180건 이상의 다운로드**가 발생한 Electron 데스크톱 앱이며, 나무학교 내부뿐 아니라 **외부 선생님들도 사용** 중이다. Google OAuth **테스트 모드는 최대 100명**의 테스트 사용자만 허용하므로, 구글 캘린더 연동 사용자가 100명을 초과할 가능성이 높다. 따라서 처음부터 **프로덕션 전환을 개발 계획에 포함**해야 한다.

#### 테스트 모드 vs 프로덕션 모드 비교

| 항목 | 테스트 모드 | 프로덕션 모드 |
|------|-----------|-------------|
| **사용자 수 제한** | 최대 100명 (수동 등록) | 제한 없음 |
| **동의 화면** | "이 앱은 확인되지 않았습니다" 경고 표시 | 경고 없이 깔끔한 동의 화면 |
| **사용자 등록** | 관리자가 이메일 직접 추가 필요 | 누구나 바로 사용 가능 |
| **토큰 만료** | 7일 후 자동 만료 (재인증 필요) | 정상 수명 (refresh token 유지) |
| **앱 신뢰도** | "확인되지 않은 앱" 딱지 | Google 검증 완료 마크 |
| **적합한 상황** | 개발/테스트 단계 | 배포/운영 단계 |

#### 프로덕션 전환 준비물 체크리스트

| # | 준비물 | 설명 | 상태 |
|---|--------|------|------|
| 1 | **개인정보처리방침 페이지 URL** | 웹에서 접근 가능한 페이지 (12절 초안 참고) | ☐ |
| 2 | **앱 홈페이지 URL** | 기존 쌤핀 랜딩페이지 활용 가능 | ☐ |
| 3 | **앱 로고** | 쌤핀 아이콘 (120×120 이상) | ☐ |
| 4 | **데모 영상** | 1~2분, 캘린더 동기화 시연 (YouTube 업로드) | ☐ |
| 5 | **scope 사용 목적 설명문** | 왜 캘린더 접근이 필요한지 설명 (영문) | ☐ |

#### 비용

- **프로덕션 전환 비용: 전부 무료** ✅
- `https://www.googleapis.com/auth/calendar`은 **Sensitive scope**에 해당
  - Sensitive scope 검증은 **무료** (유료 보안 평가 불필요)
  - 유료 보안 평가(CASA Tier 2)는 **Restricted scope**에만 해당 (예: Gmail 전체 접근)
- **Google Calendar API 자체도 무료**
  - 일일 한도: 1,000,000 쿼리
  - 쌤핀 규모에서는 한도 초과 가능성 전무

## 2. 타겟 사용자 & 시나리오

### 2.1 사용자 페르소나 (기존 확장)

#### 김선생 (중학교 국어 교사, 담임)
- 학교 PC에서 쌤핀으로 일정 관리
- **추가 니즈**: 퇴근 후 스마트폰 구글 캘린더에서 내일 일정 확인하고 싶음
- **연동 후**: 쌤핀에 입력한 "내일 3교시 공개수업" 일정이 폰 구글 캘린더에도 보임

#### 박선생 (나무학교 운영, 중학교 겸직)
- 학교 + 나무학교 이중 일정 관리
- **추가 니즈**: 구글 캘린더에 개인 일정(병원, 모임)이 있는데 쌤핀에서도 보고 싶음
- **연동 후**: 구글 캘린더의 개인 일정이 쌤핀에 "구글" 뱃지로 표시됨

## 3. 동기화 트리거 방식

### 3.1 왜 Push Notification을 사용하지 않는가?

Google Calendar Push Notification(Webhook)은 **공개 HTTPS URL + SSL 인증서**가 필수 요구사항이다. 쌤핀은 Electron 기반 데스크톱 앱이므로:

- 공개 URL을 노출할 수 없음 (사용자 PC는 NAT 뒤에 있음)
- SSL 인증서를 발급받아 유지할 서버 인프라가 없음
- 중계 서버를 두면 아키텍처 복잡도와 운영 비용이 급증

따라서 v1.0에서는 **Smart Polling** 방식을 채택한다.

### 3.2 Smart Polling 전략

| 트리거 | 동기화 방식 | 설명 |
|--------|-----------|------|
| 앱 시작 시 | incremental sync | syncToken 기반 변경분 확인 |
| 앱 포커스 복귀 | incremental sync | 백그라운드 → 포그라운드 전환 시 |
| 주기적 (기본 5분) | incremental sync | 사용자 설정 가능 (1~30분) |
| 일정 CRUD 즉시 | push only | 쌤핀 변경 → 구글에 즉시 push |
| 수동 "지금 동기화" | full incremental sync | 사용자 요청 |
| 네트워크 복귀 | 큐 flush + incremental sync | 오프라인 → 온라인 전환 시 |

### 3.3 Exponential Backoff (에러 시)

- API 에러 시: 1분 → 2분 → 4분 → 8분 → 최대 30분
- 성공 시 기본 간격으로 복귀
- Rate limit (429) 시: Retry-After 헤더 존중

### 3.4 향후 계획

서버 인프라가 구축되면(v2.0+), Google Calendar Push Notification을 도입하여 실시간 동기화를 지원할 수 있다.

## 4. 상세 UX 플로우

### 4.1 초기 연결 플로우

```
[설정] → [구글 캘린더 섹션] → "🔗 구글 캘린더 연결" 버튼 클릭
    ↓
[구글 로그인 창] (시스템 기본 브라우저에서 열림)
    → 구글 계정 선택/로그인
    → "쌤핀이 Google 캘린더에 접근하도록 허용하시겠습니까?" 동의
    ↓
[브라우저 → 쌤핀으로 자동 복귀]
    → "✅ 연결 완료! wnsdlf1212@gmail.com" 토스트 표시
    ↓
[카테고리 매핑 설정] (자동 표시, 자동 매핑 제안 포함)
    → 기존 구글 캘린더와 이름 유사도 기반 자동 매핑 제안
    → 쌤핀 카테고리별 동기화 여부 & 구글 캘린더 매핑
    → "저장" 클릭
    ↓
[첫 동기화] (자동 시작)
    → "동기화 중... (23/45)" 프로그레스
    → "✅ 동기화 완료! 45개 일정 동기화됨" 
```

**클릭 수**: 연결 버튼(1) → 구글 동의(2) → 매핑 저장(3) = **총 3클릭**

### 4.2 일상 동기화 플로우

```
[쌤핀에서 일정 추가/수정/삭제]
    ↓ (자동, 백그라운드)
[구글 캘린더 API 호출] → 구글 캘린더에 반영
    ↓
[상태바에 "☁️ 동기화됨" 표시]

---

[구글 캘린더에서 일정 변경]
    ↓ (Smart Polling: 5분 주기 / 앱 포커스 시)
[변경 감지 (incremental sync)]
    ↓
[쌤핀 로컬에 반영] → UI 업데이트
    ↓
[상태바에 "🔄 3개 일정 업데이트됨" 토스트]
```

### 4.3 연결 해제 플로우

```
[설정] → [구글 캘린더 섹션] → "연결 해제" 버튼
    ↓
[확인 다이얼로그 — 일정 보존 옵션]
    "구글 캘린더 연결을 해제하시겠습니까?
     
     구글에서 가져온 일정을 어떻게 할까요?
     ○ 모두 삭제 (기본) — 구글에서 가져온 일정을 쌤핀에서 제거합니다.
     ○ 로컬에 유지 — 일정을 쌤핀에 남기되, 더 이상 동기화하지 않습니다.
     
     ℹ️ 쌤핀 고유 일정은 어느 쪽을 선택해도 유지됩니다."
    [취소] [연결 해제]
    ↓
[토큰 삭제 + 선택에 따라 구글 일정 처리] → "연결이 해제되었습니다" 토스트
```

#### 요구사항

- **FR-GCAL-01**: "구글 캘린더 연결하기" 버튼 → OAuth 2.0 + PKCE 인증 플로우 시작.

## 7. 에러 처리 UX

### 7.1 에러 시나리오별 대응

| 에러 | 사용자에게 보이는 것 | 자동 처리 |
|------|---------------------|----------|
| **토큰 만료** | "구글 캘린더 재인증이 필요합니다 [다시 연결]" 토스트 | refresh token으로 자동 갱신 시도 → 실패 시 UI 표시 |
| **네트워크 실패** | 상태바 "⚠️ 동기화 실패" + 배너 | 30초 후 재시도, 3회 실패 시 중지 |
| **API 할당량 초과** | "잠시 후 다시 시도됩니다" 토스트 | exponential backoff로 자동 재시도 |
| **권한 거부** | "구글 캘린더 접근 권한이 필요합니다 [권한 설정]" | 설정 페이지로 안내 |
| **충돌 발생** | 충돌 해결 모달 (아래 참고) | 기본: 최근 수정 우선, 수동 선택 옵션 |
| **구글 계정 삭제/비활성화** | "구글 계정에 접근할 수 없습니다 [다시 연결]" | 연결 해제 처리 |

### 10.3 제약 사항

| 항목 | 설명 |
|------|------|
| **반복 일정** | v1.0에서는 반복 일정을 개별 인스턴스로 처리한다 (`singleEvents=true`). 반복 패턴(RRULE) 자체의 동기화는 v2.0에서 지원. |
| **Push Notification** | 데스크톱 앱 특성상 Google Calendar Push Notification 사용 불가. Smart Polling으로 대체 (3.1절 참고). |
| **프로덕션 전환 필요** | Google OAuth 테스트 모드(100명 제한)로 개발 시작 → 180명+ 사용자 규모 고려 시 프로덕션 전환 필수. 1.4절 참고. |
| **단일 계정** | v1.0에서는 1개의 구글 계정만 연결 가능. |

## 11. 용어 정의

| 용어 | 설명 |
|------|------|
| **동기화** | 쌤핀과 구글 캘린더 간 데이터를 일치시키는 과정 |
| **incremental sync** | syncToken을 이용해 변경분만 가져오는 효율적 동기화 |
| **syncToken** | 구글 API가 발급하는 토큰. 이전 동기화 이후 변경분만 조회 가능 |
| **Smart Polling** | 상황별 최적화된 폴링 전략 (앱 시작/포커스/주기적/수동) |
| **PKCE** | Proof Key for Code Exchange. OAuth 2.0 보안 강화 메커니즘 |
| **카테고리 매핑** | 쌤핀 일정 카테고리와 구글 캘린더 간 연결 관계 |
| **충돌** | 같은 일정이 양쪽에서 동시에 수정되어 어느 버전이 맞는지 결정 필요한 상태 |
| **오프라인 큐** | 네트워크 없을 때 쌓인 변경사항 목록. 온라인 복귀 시 순서대로 처리 |
| **출처 뱃지** | 일정이 쌤핀 고유인지 구글에서 온 것인지 표시하는 아이콘 (🌐G) |
| **singleEvents** | Google Calendar API 파라미터. 반복 일정을 개별 인스턴스로 펼쳐서 반환 |
| **프로덕션 모드** | Google OAuth 앱 게시 상태. 검증 완료 후 모든 사용자가 제한 없이 사용 가능 |
| **Sensitive scope** | Google API 범위 분류. 캘린더 접근 등. 프로덕션 전환 시 무료 검증 필요 |

## 12. 개인정보처리방침 초안

> Google OAuth 프로덕션 전환 신청 시 제출 필요. 쌤핀 랜딩페이지 또는 별도 페이지에 게시.

### 쌤핀 개인정보처리방침 (구글 캘린더 연동)

**1. 수집하는 정보**
- 구글 계정 이메일 주소
- 구글 캘린더 일정 데이터 (제목, 날짜, 시간, 장소)

**2. 정보 사용 목적**
- 쌤핀 앱과 구글 캘린더 간 일정 양방향 동기화

**3. 정보 저장 방식**
- 모든 데이터는 사용자의 PC에 로컬 저장
- 외부 서버로 전송하지 않음
- OAuth 토큰은 OS 키체인(safeStorage)에 암호화 저장

**4. 정보 삭제**
- 구글 캘린더 연결 해제 시 모든 토큰 및 동기화 데이터 즉시 삭제
- 앱 삭제 시 로컬 데이터 함께 삭제

**5. 제3자 제공**
- 사용자 데이터를 제3자에게 제공하지 않음
- Google Calendar API만 직접 통신

**6. 문의**

## 13. 프로덕션 전환 추천 타임라인

개발과 프로덕션 전환 심사를 **병행**하여 시간을 절약한다.

```
[개발 시작] Phase 0~1: 테스트 모드로 개발 & 테스트
     ↓ (병행)
[프로덕션 전환 신청] 개인정보처리방침 페이지 + 데모 영상 제출
     ↓ 검토 수일~수주
[Phase 2~3] 양방향 동기화 완성
     ↓
[프로덕션 승인] → 모든 사용자 즉시 사용 가능
```

### 단계별 상세

| 시점 | 작업 | 비고 |
|------|------|------|
| **Phase 0~1 병행** | 개인정보처리방침 페이지를 랜딩페이지에 추가 | 검토 시간 확보를 위해 조기 착수 |
| **Phase 1 완료 시점** | OAuth 인증 플로우가 동작하면 데모 영상 녹화 | 연결→동기화→해제 시연 (1~2분) |
| **데모 영상 완료 후** | Google Cloud Console에서 프로덕션 전환 제출 | scope 설명문 + 영상 + 개인정보처리방침 URL |
| **심사 기간 (수일~수주)** | Phase 2~3 개발 진행 | 개발과 심사 병행 |
| **프로덕션 승인** | OAuth 동의 화면 경고 제거, 사용자 수 제한 해제 | 기존 테스트 사용자 그대로 사용 가능 |

### 심사 불통과 시 대응

- Sensitive scope 검증은 대부분 **1~2주 이내** 승인
- 불통과 사유가 있으면 수정 후 재제출 가능
- 심사 중에도 테스트 모드(100명)로 내부 사용 가능
