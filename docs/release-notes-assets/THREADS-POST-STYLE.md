# 🔒 쌤핀 릴리즈 Threads 타래 글 스타일 (v1.10.2 기준 고정)

> 이 스타일은 **v1.10.2 Threads 타래(`v1.10.2/threads-post.md`)에서 사용자가 공식 채택**한 어투/구조다.
> 새 버전 릴리즈 글을 쓸 때는 반드시 이 가이드를 준수한다. 임의 변경 금지.
>
> 카드뉴스 이미지 스타일: `CARD-NEWS-STYLE.md`

---

## 1. 타래 구조

- **타래 수 = 카드 수 1:1 매핑** (카드당 1 타래)
- Thread 1(메인) ↔ 01-intro.png, Thread 2 ↔ 02-*.png, ..., Thread N ↔ 06-outro.png
- Thread 2~N은 Thread 1의 답글(reply)로 연결
- 인스타그램 캐러셀로 동일 카드 N장 교차 공유 권장

## 2. 톤·어투

- **기본**: 존댓말 (~어요, ~입니다, ~세요)
- **친근도**: 옆자리 선배 교사가 알려주는 톤 — 격식 적당, 발랄하지 않음
- **공감형 마무리**: 각 콘텐츠 타래 끝에 "실제 교실에서 어떻게 도움 되는지" 한 줄
  - ✅ "수업 중에 도구 전환하느라 버벅이던 시간이 줄어들어요"
  - ✅ "같은 내용을 두 번 입력할 일이 없습니다"
  - ✅ "한 문항씩 토론하며 진행하는 수업에 딱입니다"
- **1인칭 복수(우리·제가) 최소화**
- **기술 용어는 쓰되 설명 병기**

## 3. 길이

- **각 타래 500자 이내** (Threads 제한)
- 메인 4~6줄 / 콘텐츠 5~8줄 / 아웃트로 4~6줄

## 4. 이모지 (엄격히 절제)

| 위치 | 이모지 |
|------|--------|
| 메인 포스트 (Thread 1) | `📌` 1개 (버전 뒤) |
| 하이라이트 강조 (필요 시 1개 타래) | `⭐` 1개 |
| 아웃트로 CTA | `🔔` 또는 `🙌` 1개 |
| 그 외 콘텐츠 타래 | **0개 (금지)** |

- 이모지 연속 사용 금지
- 이모지 대체 가능한 자리는 `·` `•` `—` 등 타이포그래피 기호로

## 5. 해시태그

| 위치 | 해시태그 |
|------|----------|
| Thread 1 (메인) | `#쌤핀 #교사앱 #업데이트` (3개) |
| Thread N (아웃트로) | `#쌤핀 #SsamPin #교사도구 #교육 #EdTech` (5개, 검색 노출용) |
| 중간 타래 | **금지** |

## 6. 외부 링크

- **중간 타래 금지** (알고리즘상 노출 저하)
- **아웃트로 타래만**: `ssampin.com` (프로토콜 생략)

## 7. 타래별 템플릿

### Thread 1 — 메인 포스트
```
쌤핀 v{VERSION} 업데이트 📌 ({YYYY.MM.DD} 릴리즈)

이번엔 '{한 줄 컨셉}' 하는 마음으로 {N}가지를 손봤어요.

• {하이라이트 1}
• {하이라이트 2}
• {하이라이트 3}
• {하이라이트 4}

아래 타래에서 하나씩 소개할게요 👇

#쌤핀 #교사앱 #업데이트
```

### Thread 2~(N-1) — 콘텐츠 타래
```
{번호}. {기능명} — {한 줄 후킹 또는 질문}

{설명 2~3 문장. 기능 동작 방식과 실제 활용 시나리오.
불릿(·)으로 세부 항목 나열 가능.}

{현실 공감 마무리 한 줄. 교실·수업 장면에 비추어.}
```

세부 기능 나열 시 예시:
```
• 글자 크기: 작게 / 보통 / 크게 / 아주 크게 4단계
• 이미지 첨부: 붙여넣기·드래그앤드롭 (PNG·JPEG·WebP)
• 링크 삽입: 툴바 링크 버튼으로 http·https·mailto·tel 지원
```

수업에서 써보시고 피드백 주시면 다음 버전에 바로 반영하겠습니다. 감사합니다 🙌

## 8. 번호 매김

- 콘텐츠 타래 기능 앞에 `1.` `2.` `3.` 순서 번호
- 카드 번호와는 별개 (카드 1=인트로이므로 기능 번호는 1부터 시작)

## 9. 구두점

- 강조·분리에는 `—` (em dash). `-` (hyphen)이나 `–` (en dash) 금지
- 세부 나열은 `·` 또는 `•` 사용
