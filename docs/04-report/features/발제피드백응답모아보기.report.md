### 2.2 Phase 1에서 발견·수정된 주요 개선 사항

#### 1. Portal(createPortal) 필수 발견 — 모달 containing block 버그 방지 ⭐

**상황:** Modal 모드의 SpreadsheetView를 브라우저에서 테스트할 때, ToolLayout 계층의 `transform` 스타일이 있으면 모달이 시각적으로 잘리거나 오프셋되는 현상 발생.

**원인:** CSS 사양상 `position: fixed`의 containing block은 viewport가 아니라 특정 CSS 속성(`transform`, `filter` 등)을 가진 조상으로 변환됨.

**해결:** `createPortal(..., document.body)`로 DOM 트리에서 body 직속으로 이동시켜 조상 영향 차단.

```typescript
if (source.mode === 'modal') {
  return createPortal(
    <div className="fixed inset-0 z-[100] flex items-center justify-center bg-black/60 ...">
      <div className="h-[90vh] w-full max-w-6xl bg-sp-bg ...">
        {/* 모달 내용 */}
      </div>
    </div>,
    document.body,  // ← 필수
  );
}
```

**Design 역반영:** v0.3 문서에 "§3.1 Modal 모드: `createPortal(document.body)` 필수 명시 + containing block 트랩 설명" 추가.

#### 2. 라이트 테마 호환 보정 — 투명도 변형 금지

**상황:** 프로젝트 기본 테마가 Notion 스타일 라이트(`sp-bg: #ffffff`). 구현 초기에 막대 그래프를 `bg-sp-accent/70` (투명도 70%)로 표현하려 하니 밝은 배경 위에서 대비 부족으로 거의 안 보임.

**해결:** 모든 차트·그래프·막대를 **불투명 accent**(`bg-sp-accent`)만 사용하도록 통일. hover 상태만 `hover:bg-sp-accent/80` 등으로 변형.

**Design 역반영:** v0.3 "§3.1 디자인 토큰" 항목에 "투명도 변형(`/20`, `/70`)은 라이트 테마에서 대비 저하 → 불투명 accent + hover 변형만 사용" 가이드 추가.

#### 3. Excel Sheet 2 헤더 2열 분리 — 정렬 및 필터 편의성

**설계:** Sheet 2의 "응답자 라벨 + 제출 시각"을 1열로 기술.

**구현:** 실제로는 A열(응답자)과 B열(제출 시각)을 분리하여, Excel 정렬·필터 시 더 유연하게 대응.

**Design 역반영:** v0.3 "§4.2 Sheet 2 테이블"을 3열 분리(응답자 / 제출 시각 / Q1…)로 명시.

### 2.3 Phase 2에서 발견·수정된 주요 개선 사항

#### 1. `handleReset`/`handleFinish`의 `showFeedbackWall` 상태 리셋 누락

**상황:** 피드백 월 토글 후 설문을 완료하고 다시 new 설문을 시작하려 할 때, `showFeedbackWall` state가 리셋되지 않아 피드백 월 뷰가 남아있게 됨.

**해결:** `handleReset()` 및 `handleFinish()` 핸들러에 `setShowFeedbackWall(false)` 추가.

```typescript
// ToolMultiSurvey.tsx L1104, L1241
handleReset() {
  // ...
  setShowFeedbackWall(false);  // ← 추가
}

handleFinish() {
  // ...
  setShowFeedbackWall(false);  // ← 추가
}
```

#### 2. 피드백 월의 `toggle` 헬퍼 useCallback deps 이슈

**상황:** `toggleHighlight` / `togglePin` 헬퍼 함수가 `useCallback` 의존성 배열에 `set` / `setSet`을 포함하지 않아 eslint 경고 및 스테일 클로저 위험.

**해결:** 함수형 업데이트(`prev => ...`) 패턴으로 리팩터하여 클로저 제거.

```typescript
// 수정 전
const toggleHighlight = useCallback((cardId: string) => {
  setHighlighted(new Set([...highlighted, cardId]));  // ← deps 누락
}, []); // ❌

// 수정 후
const toggleHighlight = useCallback((cardId: string) => {
  setHighlighted(prev => {
    const newSet = new Set(prev);
    if (newSet.has(cardId)) newSet.delete(cardId);
    else newSet.add(cardId);
    return newSet;
  });
}, []); // ✅ deps 간결
```

**발견:** `toolResultAggregation.ts`, `toolResultSerialization.ts`를 순수 함수로 domain에 격리한 결과:
