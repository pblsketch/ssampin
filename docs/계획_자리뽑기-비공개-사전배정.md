# 자리 뽑기 — 공개/비공개 사전 배정 (최종 설계)

> 기준일: 2026-04-21
> 관련 피드백: 2026-04-21 14:58 KST · 도구 > 자리 뽑기 · 기능 개선

---

## 요구사항

두 가지 서로 다른 교사 욕구를 **모두** 충족해야 한다.

1. **공개 시나리오** — 학생과 함께 "얘들아 앞자리 줄게" 식으로 고정을 정하는 경우. 고정 사실을 학생도 앎.
2. **비공개 시나리오** — 교사가 혼자 말썽꾸러기를 떨어뜨려 놓는 경우. **설정 화면조차 학생에게 노출되면 안 됨**.

---

## 핵심 설계: 두 개의 설정 출입구, 하나의 데이터 모델, 하나의 뽑기 로직

### 출입구 A — 공개 경로 (기존 유지)
- 위치: **자리 뽑기 > 좌석 설정 > 📌 학생 고정 탭**
- 그대로 📌 노출, "고정 N명" 요약 표시. 학생도 봐도 됨.

### 출입구 B — 비공개 경로 (신규)
- 위치: **설정(/settings) > 🛠️ 도구 탭 > 자리 뽑기 · 비공개 사전 배정**
- 학생이 열지 않는 다른 창. 여기서 지정한 고정은 자리 뽑기 setup 화면에 **전혀 노출되지 않음**(숫자·그리드·요약 모두).
- 영구 저장 (세션 간 유지). Repository: `ISeatPickerConfigRepository` / 파일: `seat-picker-config.json`.
- scope별 분리: `homeroom` / `tc-${teachingClassId}`.

### 데이터 모델
```ts
// 공개 고정: 기존 fixedStudents (뽑기 세션 state, 파일 저장 X)
//   — 자리 뽑기 setup 탭에서 즉석 지정.
// 비공개 고정: SeatPickerConfig.privateAssignments (파일 저장)
//   — 설정 > 도구에서 관리.
```

### 뽑기 로직 — 같은 엔진, 학생별 다른 경로
- **공개 고정 학생**: 참여자(`studentOrder`)에서 제외 + 시작 순간 `assignments`에 확정 배정 + 📌 표시.
- **비공개 고정 학생**: 참여자에 **포함** + 해당 학생 차례에 카드를 클릭하면 **`shuffledSeats` 스왑**으로 사전 지정 좌석에 도달 + 📌 표시 없음.
- 둘이 섞여도 정상 작동. 한 뽑기에서 공개 2명 + 비공개 2명 가능.

## 학생이 보는 숫자가 정직해지는 이유

| 지표 | 이전 | 이제 |
|---|---|---|
| 카드 수 | `총좌석 - 고정 - 빈자리` → 티남 | `총좌석 - 공개고정 - 빈자리` = 참여자 수 (비공개 좌석도 카드 풀에 포함) |
| 대기 목록 | 고정 학생 누락 | 공개 고정만 제외. 비공개 고정 학생은 **정상 포함** |
| 진행바 초기값 | `N/M` (고정만큼 앞서 있음) | `0/참여자수` (공개 고정도 배정 포함이지만 별도 표시 대상이라 자연스러움) |
| 시작 시 채워진 칸 | 공개+비공개 모두 | **공개만**. 비공개는 빈 칸으로 보임 |
| 설정 화면의 📌·"고정 N명" | 둘 다 | **공개만**. 비공개는 완전 invisible |

## 검증 가드
- 같은 좌석에 공개·비공개가 동시 설정되면 `publicVsPrivateCollision` → 빨간 경고, 뽑기 시작 차단.
- 비공개 고정 학생이 `activeStudents`에 존재하지 않으면(졸업/탈퇴 등) 스왑 로직이 안전하게 실패하고 일반 뽑기로 진행.

## 수용 기준
- **공개 경로**: 자리 뽑기 좌석 설정에서 학생 고정 → 시작 → 기존과 동일하게 📌 노출로 진행.
- **비공개 경로**: 설정 > 도구에서 학생 지정 → 자리 뽑기 setup에 아무 변화 없음(고정 N명·📌 없음) → 시작 → 해당 학생이 일반 학생처럼 카드 뽑고, 카드가 "우연히" 지정 좌석에 떨어짐 → 완료 화면/결과표에도 📌 없음.
- **혼합**: 공개 2 + 비공개 1 설정 후 진행. 공개는 📌로 드러나고, 비공개는 자연스럽게 숨겨진 채 진행.
- 타입 체크 통과 (`npx tsc --noEmit` exit=0).
