## 1. 제품 개요

### 1.1 제품명

**쌤핀 모바일** (Ssampin Mobile)

### 1.2 목적

데스크톱 전용인 쌤핀(v0.4.8)의 핵심 기능을 교사가 **교실 어디서든 스마트폰으로** 빠르게 확인·기록할 수 있도록 하는 Progressive Web App.

### 1.3 타겟 사용자

- 초·중·고 교사 (담임교사, 교과교사, 관리자/부장)
- 데스크톱 쌤핀을 이미 사용 중이거나, 모바일 단독으로 시작하는 교사

### 1.4 핵심 가치

| 가치 | 설명 |
|------|------|
| **즉시성** | 스마트폰에서 3초 안에 현재 수업·출결 확인 |
| **동기화** | Google Drive를 통해 데스크톱 ↔ 모바일 데이터 실시간 동기화 |
| **비용 $0** | 앱스토어 없이 PWA 설치, Vercel 무료 배포, 서버 비용 없음 |
| **보안** | E2E 암호화로 학생 개인정보 보호, Google Drive에도 암호화된 상태로 저장 |
| **오프라인** | Service Worker 캐싱으로 네트워크 없이도 기본 기능 사용 가능 |

### 1.5 핵심 결정사항

1. **Google Drive를 동기화 백엔드로 활용** — 데스크톱 v0.4.2+에 이미 구현된 `SyncToCloud`/`SyncFromCloud` UseCase 재사용
2. **PWA (Progressive Web App)** — 앱스토어 없이 홈 화면에 추가
3. **브랜치 분리** — `feature/mobile`에서 개발, `domain/` + `usecases/` 공유
4. **클린 아키텍처** — domain + usecases 100% 재사용, infrastructure만 플랫폼별 분기
5. **비용 $0/월** — Vercel 배포

## 2. 사용 시나리오

### 시나리오 1: 아침 등교 — "오늘 뭐 하지?"

> 김 선생님은 출근 중 버스에서 스마트폰으로 쌤핀 모바일을 연다.
> "오늘" 허브에서 1교시 수학(3-2반), 2교시 수학(3-5반)… 시간표를 한눈에 확인한다.
> 급식 메뉴도 스크롤 없이 바로 보인다. "오, 오늘 불고기네."

### 시나리오 2: 조회 시간 — "담임 출결 체크"

> 조회 시간, 김 선생님은 교탁에서 스마트폰을 꺼내 "오늘" 허브의 "우리 반 출결" 카드를 탭한다.
> 학생 명단이 큰 버튼으로 나열되고, 결석인 3번 이민수를 탭하여 '결석'으로 변경.
> 44px 이상의 큰 터치 타겟으로 실수 없이 빠르게 체크.

### 시나리오 3: 수업 중 — "이 반 출결"

> 3교시 시작, 3-5반 교실에 들어간 김 선생님.
> "오늘" 허브에서 "수업 출결 · 3교시" 카드를 탭.
> 3-5반 학생 명단이 뜨고, 결석 2명, 지각 1명을 빠르게 기록.

### 시나리오 4: 쉬는 시간 — "할 일 확인"

> 쉬는 시간에 "할 일" 탭을 열어 오늘 마감인 성적 입력 업무를 확인.
> 완료 체크박스를 탭하여 처리. 데스크톱에 자동으로 동기화.

### 시나리오 5: 점심시간 — "메모 작성"

> 동료와 점심을 먹으며 다음 주 학부모 상담 관련 아이디어를 "더보기 → 메모"에 빠르게 기록.
> 퇴근 후 데스크톱에서 이어서 정리.

### 시나리오 6: 방과 후 — "일정 확인"

> 일정 탭에서 내일 교직원 회의가 있는 것을 확인.
> "+" 버튼으로 간단한 준비사항 일정을 추가.

### 시나리오 7: 신규 사용자 — "첫 설치"

> 동료 추천으로 m.ssampin.com에 접속. PWA 설치 안내를 따라 홈 화면에 추가.
> 온보딩에서 학교명, 학년/반, 역할(담임+교과)을 선택.
> Google 로그인 후 데스크톱 데이터가 바로 동기화.

## 4. 화면별 상세 명세

### 4.1 탭바 구조 (확정)

```
┌──────────────────────────────────────────────────┐
│  [오늘]   [일정]   [학생]   [할 일]   [더보기]  │
└──────────────────────────────────────────────────┘
```

하단 고정 탭바, Material Symbols 아이콘 사용.

| 탭 | 아이콘 | 기본 뷰 |
|----|--------|---------|
| 오늘 | `today` | 통합 허브 (시간표 + 출결 + 급식) |
| 일정 | `event_note` | 월간/주간 캘린더 |
| 학생 | `people` | 좌석뷰 + 명단 |
| 할 일 | `check_circle` | 체크리스트 |
| 더보기 | `more_horiz` | 메모, 설정, 동기화 |

### 4.2 "오늘" 허브

모바일의 메인 화면. 세로 스크롤 카드 레이아웃.

#### 카드 1: 현재 수업 (최상단, 가장 크게)

```
┌─────────────────────────────────────┐
│  📚 3교시 · 수학                     │
│  ━━━━━━━━━━━━━━━━●━━━━━ 35분 남음    │
│  3학년 2반 교실                       │
│                                      │
│  ▸ 다음: 4교시 국어 (3-5반)          │
└─────────────────────────────────────┘
```

- **데이터 소스:** `teacher-schedule` (교사 시간표) + `Settings.periodTimes`
- **진행 바:** 현재 시각 기준 교시 진행률 실시간 계산
- **수업 없는 시간:** "수업 없음 · 다음: N교시 과목" 표시
- **방과 후:** "오늘 수업이 끝났습니다" 표시

#### 카드 2: 담임 출결 — "우리 반 출결" (담임 역할 시 표시)

```
┌─────────────────────────────────────┐
│  🏫 우리 반 출결                     │
│  출석 28 · 결석 1 · 지각 1          │
│                              [체크] │
└─────────────────────────────────────┘
```

- **표시 조건:** `Settings.teacherRoles`에 `'homeroom'` 포함 시
- **[체크] 탭 시:** 담임 출결 체크 화면으로 이동
- **데이터 소스:** `attendance` (AttendanceRecord, classId = 담임반)
- **출결 상태:** 출석 / 결석 / 지각 / 조퇴 / 외출 (5종)

#### 카드 3: 수업 출결 — "수업 출결 · N교시" (교과/담임 역할 시 표시)

```
┌─────────────────────────────────────┐
│  📋 수업 출결 · 3교시                │
│  3-2반 수학 · 출석 30 · 결석 2      │
│                              [체크] │
└─────────────────────────────────────┘
```

- **표시 조건:** 현재 교시에 수업이 있을 때
- **데이터 소스:** `attendance` (AttendanceRecord, period = 현재 교시)
- **출결 상태:** 출석 / 지각 / 결석 (3종)

#### 카드 4: 급식 메뉴

```
┌─────────────────────────────────────┐
│  🍚 오늘 급식                        │
│  잡곡밥 · 불고기(5,6,13)            │
│  된장찌개(5,6) · 깍두기(9)          │
│  693.2 Kcal                          │
└─────────────────────────────────────┘
```

- **데이터 소스:** `MealCache` (NEIS 급식 API)
- **알레르기 번호** 표시
- 급식 없는 날: "급식 정보가 없습니다" 표시

#### 역할별 카드 표시 규칙

| 역할 조합 | 카드 1 | 카드 2 | 카드 3 | 카드 4 |
|-----------|--------|--------|--------|--------|
| 담임만 | 현재 수업 | 우리 반 출결 | — | 급식 |
| 교과만 | 현재 수업 | — | 수업 출결 | 급식 |
| 담임 + 교과 | 현재 수업 | 우리 반 출결 | 수업 출결 | 급식 |
| 관리자 | 현재 수업 | — | — | 급식 |

### 4.3 출결 체크 화면

"우리 반 출결" 또는 "수업 출결" 카드의 [체크] 탭 시 진입.

#### 담임 출결 체크

```
┌─────────────────────────────────────┐
│  ← 우리 반 출결 · 2025-07-14       │
├─────────────────────────────────────┤
│  [출석] [결석] [지각] [조퇴] [외출] │  ← 필터/일괄 모드 전환
├─────────────────────────────────────┤
│  1  김민수  ●출석                    │  ← 44px 높이 행
│  2  이영희  ●출석                    │
│  3  박준호  ○결석                    │  ← 탭하여 상태 변경
│  4  최수진  ●출석                    │
│  ...                                 │
│  28 정하은  △지각                    │
└─────────────────────────────────────┘
```

- **터치 타겟:** 각 학생 행 최소 44px 높이
- **상태 변경:** 학생 행 탭 → 순환 (출석 → 결석 → 지각 → 조퇴 → 외출 → 출석)
- **데이터:** `AttendanceRecord` (classId = 담임반, date = 오늘, period = 0 또는 전체)
- 학생 목록: `Student[]` (students 파일)

#### 수업 출결 체크

```
┌─────────────────────────────────────┐
│  ← 수업 출결 · 3교시 · 3-2반 수학  │
├─────────────────────────────────────┤
│  [출석] [지각] [결석]               │
├─────────────────────────────────────┤
│  1  김민수  ●출석                    │
│  2  이영희  ●출석                    │
│  3  박준호  ○결석                    │
│  ...                                 │
└─────────────────────────────────────┘
```

- 수업반(TeachingClass) 학생 사용
- **상태:** 출석 / 지각 / 결석 (3종)
- **데이터:** `AttendanceRecord` (classId = 수업반 ID, period = 현재 교시)

### 4.4 일정 탭

- **월간 뷰:** 달력 그리드, 날짜별 일정 도트 표시
- **주간 뷰:** 7일 리스트, 시간대별 일정
- **일정 추가:** FAB(+) 버튼 → 바텀시트에서 제목, 날짜, 카테고리 입력
- **카테고리:** 기본 5종 (`DEFAULT_CATEGORIES`: 학교/학급/부서/나무학교/기타) + 커스텀
- **데이터 소스:** `SchoolEventsData`

### 4.5 학생 탭

#### 좌석뷰

- `SeatingData` 기반 좌석 배치를 읽기 전용으로 렌더링
- 교탁 기준 학생 시점 / 교사 시점 전환 (`seatingDefaultView`)
- 짝꿍 모드(`pairMode`) 지원
- **터치:** 학생 셀 탭 → 간단 정보 팝오버 (이름, 번호)

#### 명단 뷰

- 학생 목록 (`Student[]`) 리스트 표시
- 번호, 이름, (결번 표시)
- 검색 기능

### 4.6 할 일 탭

- `TodosData` 기반 체크리스트
- 완료/미완료 토글 (큰 체크박스)
- 카테고리별 필터 (`TodoCategory`: 수업/업무/학생/회의/기타)
- 우선순위 표시 (high/medium/low/none)
- 마감일 표시
- 하위 작업(`SubTask`) 표시 + 토글
- 새 할 일 추가 (간단 입력)

### 4.7 더보기 탭

| 메뉴 | 설명 |
|------|------|
| 메모 | `MemosData` 기반, 리스트 형태 (포스트잇 아닌 리스트). 새 메모 작성/편집 |
| 설정 | 테마, 역할, 학교 정보 변경 |
| 동기화 | Google Drive 연결 상태, 수동 동기화, 마지막 동기화 시각 |
| 정보 | 앱 버전, 피드백 링크 |

## 5. 동기화 설계

### 5.1 아키텍처 개요

```
┌─────────────┐     ┌──────────────────┐     ┌──────────────┐
│  데스크톱    │     │   Google Drive   │     │  모바일 PWA  │
│  (Electron)  │ ←→  │  "쌤핀 동기화"   │ ←→  │  (Browser)   │
│              │     │  폴더           │     │              │
│ Electron     │     │                  │     │ IndexedDB    │
│ StorageAdapter│     │ manifest.json   │     │ StorageAdapter│
└─────────────┘     │ settings.json    │     └──────────────┘
                     │ students.json    │
                     │ attendance.json  │
                     │ events.json      │
                     │ memos.json       │
                     │ todos.json       │
                     │ ...              │
                     └──────────────────┘
```

### 5.2 동기화 파일 목록

기존 `SYNC_FILES` 상수 그대로 사용:

```typescript
const SYNC_FILES = [
  'settings', 'class-schedule', 'teacher-schedule', 'students',
  'seating', 'events', 'memos', 'todos', 'student-records',
  'bookmarks', 'surveys', 'seat-constraints', 'teaching-classes', 'dday',
  'curriculum-progress', 'attendance', 'consultations',
] as const;
```

### 5.3 동기화 흐름

1. **업로드 (SyncToCloud):**
   - 로컬 데이터 → JSON 직렬화 → SHA-256 체크섬 계산
   - 체크섬 비교, 변경분만 Drive에 업로드
   - 매니페스트(`manifest.json`) 갱신

2. **다운로드 (SyncFromCloud):**
   - Drive 매니페스트 조회 → 로컬 매니페스트와 비교
   - 체크섬 다르면 다운로드 → 로컬 저장
   - 충돌 시 정책에 따라 처리

3. **충돌 해결:**
   - `conflictPolicy: 'latest'` → 최신 타임스탬프 우선
   - `conflictPolicy: 'ask'` → 사용자에게 선택 요청 (로컬/리모트)

### 5.4 오프라인 동작

- IndexedDB에 모든 데이터 로컬 저장
- 네트워크 복귀 시 자동 동기화 트리거
- 오프라인 상태 표시 (탭바 상단 배너)

## 6. 비기능 요구사항

### 6.1 성능

| 지표 | 목표 |
|------|------|
| 초기 로딩 (LCP) | < 2초 (3G 환경) |
| 탭 전환 | < 100ms |
| 출결 체크 반응 | < 50ms |
| 동기화 완료 | < 5초 (일반적 데이터 크기) |
| PWA 설치 후 재방문 | < 1초 |

### 6.2 보안

- **Google OAuth 2.0 + PKCE** — 브라우저 환경에서 안전한 인증
- **E2E 암호화** — 학생 개인정보(이름, 연락처, 학부모 연락처) 암호화 후 Drive 저장
- **토큰 관리** — Access Token은 메모리, Refresh Token은 IndexedDB 암호화 저장
- **PIN 보호** — 기존 `PinSettings` 활용, 보호 기능별 잠금

### 6.3 접근성

- 터치 타겟: 최소 44×44px (WCAG 2.5.5)
- 색상 대비: WCAG AA 이상
- 스크린 리더 지원: ARIA 레이블
- 키보드 네비게이션 지원

### 6.4 오프라인

- Service Worker: 앱 셸 + 정적 자산 프리캐시
- IndexedDB: 모든 데이터 로컬 저장
- 배경 동기화: `navigator.serviceWorker.ready` + `SyncManager`

### 6.5 호환성

- iOS 15+ Safari (PWA 지원)
- Android Chrome 90+
- Samsung Internet 15+
- 반응형: 320px ~ 428px (모바일), 768px+ (태블릿은 추후)
