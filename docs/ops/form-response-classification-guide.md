# Google Form 베타 피드백 수동 분류 가이드

> **목적**: v1.10.3 협업 보드 & 발제피드백 응답 모아보기 통합 Google Form(`forms.gle/o1X4zLYocUpFKCzy7`)의 응답을 **분류 자동화 없이 수동으로 라벨링**하기 위한 워크플로우 및 기준을 정의한다.
>
> **적용 시점**: v1.10.3 베타 배포(2026-04-20) 이후, Phase 1b 착수(약 2~4주) 전
>
> **결정 이력**: Phase 1b Plan §8 (2026-04-21) — 드롭다운·Apps Script 자동 라벨링 보류, "C안 수동" 채택

---

## 1. 목적 — 왜 수동 분류인가

**배경**: v1.10.3부터 ToolCollabBoard 베타 배너·FeedbackWallView 베타 배너 2곳이 동일한 Google Form으로 통합되었다. 응답은 두 가지 기능에서 섞여 들어오고, Phase 1b 착수 시점에 어떤 피드백을 먼저 반영할지 결정해야 한다.

**원래 계획했던 선택지**:
- **A안 (드롭다운)**: Form에 "어느 기능?"을 선택 항목으로 추가 → 응답자가 선택
- **B안 (Apps Script)**: Google Sheet 함수로 자동 분류 (NLP 또는 키워드 매칭)
- **C안 (수동)**: 응답이 들어온 후 Claude/사용자가 스프레드시트 직접 검토 → 라벨 기입

**C안 채택 이유**:
1. 응답 초기에는 건수가 적음(목표 3~5건)
2. 자동 분류 시 오분류 비용 > 수동 분류 비용
3. 응답 텍스트를 직접 읽으며 새로운 제안 패턴 발견 가능
4. Phase 1b 착수 조건(§2)이 나올 때까지 "버그 vs 개선요청" 구분만 명확하면 충분

## 2. 응답 접근 방법 — Google Form 스프레드시트 위치

**Google Form 응답 스프레드시트**:
- 위치: [TBD — Form 소유자 pblsketch 확인 필요]
- 매체: Google Sheets (응답 탭 → 스프레드시트로 자동 링크됨)
- 접근: Form 소유자(pblsketch) 또는 편집 권한자만 열 수 있음

**스프레드시트 기본 구조** (자동 생성됨):
| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| Timestamp | Email | 기능 선택 | 피드백 내용 | **category** | **priority** | **notes** |
| 2026-04-21 10:30 | teacher@example.com | — | 화면이 검은색만 나와요 | [여기 입력] | [여기 입력] | [여기 입력] |

**작업 규칙**:
- **E열(category)**: 네 가지 분류 중 하나 입력 (§3 참조)
- **F열(priority)**: bug-hotfix면 `hotfix`, 나머지는 비우거나 `backlog` 입력
- **G열(notes)**: 개인정보 또는 추가 맥락 (선택, 필요시만 입력)
- 색상 코드: E열 `bug-hotfix` = 빨강, `collab-board-ux` = 파랑, `multi-survey-ux` = 녹색, `other-feature` = 회색 (가독성 향상)

**직접 작업 단계**:
1. Form 응답 스프레드시트 열기
2. C열 끝 행부터 E열 기준으로 한 줄씩 검토
3. 응답 내용(D열)과 timestamp(A열)를 확인
4. E, F, G열 기입
5. 매주 목요일 오전에 일괄 처리 (또는 응답 5건 단위)

## 3. 라벨 카테고리 (4개) & 라벨링 기준

### 3.1 `bug-hotfix` — 현재 베타에서 즉시 수정 필요한 버그

**정의**: 협업 보드·발제피드백 기능이 **정상 동작하지 않아 사용 불가능**한 상태.

**포함 기준**:
- 앱 크래시
- "화면이 검은색만 나온다" / "입력 버튼이 응답 안 함"
- "저장이 안 된다" (저장 버튼 클릭 후에도 데이터 유실)
- "불러올 수 없다" (파일 열기 실패, 로딩 무한 루프)
- 학생 이름 입력 후 페이지 팅김 / 렉

**예시**:
```
"학생 이름 5명 입력하고 확인 눌렀는데 화면이 먹통입니다. 어떻게 해야 하나요?"
→ bug-hotfix (학생 이름 입력 → 확인 → 화면 정지 = 차단 버그)

"모둠 그려서 저장했는데 다음날 열어보니 아무것도 없어요"
→ bug-hotfix (데이터 손실 = 심각 버그)
```

**우선순위**: 즉시 처리. Phase 1b 기다리지 말고 **별도 hotfix 릴리즈** 또는 **v1.10.3.1** 패치로 배포.

---

### 3.2 `collab-board-ux` — 협업 보드 UX 개선 요청

**정의**: 협업 보드(ToolCollabBoard) 기능은 동작하지만, **Jamboard 수준의 교육용 UX에 필요한 개선사항**.

**포함 기준**:
- 그리기 도구(펜, 지우개, 도형 등) 색깔·굵기 옵션 추가
- 실행 취소(Undo)/다시 실행(Redo) 요청
- 배경색·그리드 표시 옵션
- 텍스트 삽입 기능
- 스티키 노트 기능
- 페이지 추가/삭제
- 팜 리젝션 (손가락 입력 무시)
- 학생 입장 화면 개선

**예시**:
```
"펜 색깔이 검은색, 빨강, 파랑 3가지뿐인데 초록색도 있으면 좋겠어요"
→ collab-board-ux (색깔 옵션 추가 = Phase 1b 범위)

"그린 것을 실수로 지웠는데 복구 버튼이 없어서 처음부터 다시 그렸어요"
→ collab-board-ux (Undo 기능 = Phase 1b 계획 §3.2)

"태블릿으로 그리는데 손가락이 화면 닿으면 줄이 튀어요"
→ collab-board-ux (팜 리젝션 = Phase 1b 입력 품질 개선)
```

**우선순위**: Phase 1b 범위 및 우선순위. Plan §3.1~§3.3에 반영 (3건 이상 수집 후).

---

### 3.3 `multi-survey-ux` — 발제 피드백 응답 모아보기 UX 개선

**정의**: 발제피드백 응답 모아보기(FeedbackWallView) 기능 개선요청. **별도 세션에서 담당하는 기능**.

**포함 기준**:
- 응답 검색/필터 옵션 추가
- 응답 정렬 순서 변경
- 응답 묶음(그룹화) 기능
- 학생 이름 필터
- 응답 통계 추가
- 내보내기(CSV/Excel) 형식 변경

**예시**:
```
"응답을 날짜순으로 정렬할 수 없나요? 지금은 받은 순서대로만 보여요"
→ multi-survey-ux (정렬 기능 = 별도 세션)

"학생이 100명인데 특정 이름으로 응답을 검색할 방법이 있나요?"
→ multi-survey-ux (검색 기능 = 별도 세션)
```

**우선순위**: 별도 세션에서 관리. 현재 Phase 1b에는 포함 안 함.

---

### 3.4 `other-feature` — 다른 기능 요청 / 향후 로드맵

**정의**: 협업 보드도 발제피드백도 아닌 **다른 기능 요청** 또는 **원본 협업 보드와 무관한 제안**.

**포함 기준**:
- 쌤핀의 다른 기능(시간표, 좌석배치, 메모 등) 개선요청
- 협업 보드의 "완전히 새로운 기능" (현재 계획에 없는 것)
- 설정/로그인/계정 관련 이슈
- 기타 피드백 (일반 칭찬, 건의사항 등)

**예시**:
```
"시간표 창에서도 현재 교시를 큰 글씨로 표시해 주세요"
→ other-feature (협업 보드 무관)

"협업 보드에서 동영상 첨부는 안 되나요?"
→ other-feature (현재 계획 범위 밖)

"좋은 앱 만들어주셔서 감사합니다!"
→ other-feature (칭찬)
```

**우선순위**: 향후 로드맵 또는 2순위 이하. 현재 Phase 1b에 포함 안 함.

## 4. 우선순위 결정 규칙

### 4.1 `bug-hotfix` — 즉시 처리 경로

- **발견 순간**: 응답 1건만 있어도 즉시 조치
- **처리 방식**: 별도 hotfix 릴리즈 또는 다음 버전에 긴급 반영
- **예시 일정**:
  - 2026-04-22 버그 리포트 수집
  - 2026-04-22 ~ 2026-04-23 재현 & 수정
  - 2026-04-23 또는 2026-04-24 v1.10.3.1 배포 (Electron auto-update)

### 4.2 `collab-board-ux` — Phase 1b 착수 시점에 재우선화

- **수집 기준**: 최소 3건 이상
- **처리 방식**: Phase 1b Plan §3.1~§3.3 범위와 우선순위 비교 후, 최대 상위 5건까지 포함
- **일정 예시**:
  - 2026-04-28 ~ 2026-04-30 피드백 수집 (최소 3~5건)
  - 2026-05-01 우선순위 재조정 (Plan §3 재작성)
  - 2026-05-05 Phase 1b 개발 착수

### 4.3 `multi-survey-ux` / `other-feature` — 별도 세션 이월

- 현재 Phase 1b에는 포함 안 함
- 다음 Feature 세션 시 로드맵 반영
- 건수 누적 보유 (분기별 검토)

## 5. 작업 흐름 — 주간 분류 프로세스

### 5.1 매주 1회 분류 작업

**주기**: 매주 목요일 오전 10시 (또는 응답 5건 단위)

**체크리스트**:
1. [ ] Google Form 응답 스프레드시트 열기
2. [ ] 미분류 응답(E열 빈 행) 확인 (필터 > "Blank rows" 사용)
3. [ ] 각 응답 읽기:
   - D열(피드백 내용) 정독
   - A열(timestamp) 및 B열(이메일) 맥락 확인
   - 협업 보드 vs 발제피드백 응답인지 판단
4. [ ] E열 라벨 기입 (§3 카테고리 중 하나)
5. [ ] F열 우선순위 기입:
   - `bug-hotfix` → `hotfix`
   - 나머지 → 비우거나 `backlog`
6. [ ] G열 메모 (선택, 개인정보 마스킹 필수)
7. [ ] 저장 및 완료

### 5.2 분류 후 즉시 실행 항목

**`bug-hotfix` 1건 이상 발견 시**:
- Slack/Discord에 알림 (pblsketch에 @mention)
- 제목: `[BUG-HOTFIX] <응답 한 줄 요약>`
- 내용: 응답 전문 + timestamp + 이메일 (필요시)

**`collab-board-ux` 3건 도달 시**:
- 현재까지 수집된 3건을 Phase 1b Plan §3 으로 복사
- "우선순위 재조정 예정일" 일정 표시

## 6. 개인정보 및 보안 주의사항

### 6.1 응답에 포함될 수 있는 민감 정보

- 학생 이름 (예: "저희 반 홍길동이가...")
- 교사 이메일 (B열, Google Form 로그인한 계정)
- 학교명 / 학년 / 반 이름
- 특정 학생의 행동/성적 언급

### 6.2 금지 사항

- [ ] 스프레드시트를 GitHub / Notion / 이메일로 공유 금지
- [ ] 개인 클라우드 드라이브에 다운로드 금지
- [ ] 스크린샷 캡처 금지 (학생 이름 노출 위험)
- [ ] 응답 내용을 공개 채널(Slack, Discord)에 붙여넣기 금지

### 6.3 보호 조치

- Google Form & 스프레드시트: "편집 권한자만" 설정 유지
- 필요한 경우만 G열에 메모 기입 (학생 이름 명시 금지)
- 1개월 이상 지난 개인정보는 G열 비우기 또는 마스킹 ("학생A", "교사B" 식)

## 7. 다음 업그레이드 조건 — 자동 분류로 전환

### 7.1 지표

| 지표 | 현재 | 업그레이드 트리거 |
|------|------|------------------|
| 월 응답 건수 | ~0.5건 | **20건 초과** |
| 분류 오류율 | 0% | **5% 이상** |
| 분류 소요 시간 | ~5분/회 | **30분 초과/회** |

### 7.2 업그레이드 방식 (B안 또는 A안)

**B안 추천** (자동 분류 우선):
- Google Sheet Apps Script로 키워드 매칭 함수 작성
- "크래시", "안 됨", "검은색", "팅김" → `bug-hotfix`
- "색깔", "Undo", "펜", "팜 리젝션" → `collab-board-ux`
- "정렬", "필터", "검색", "통계" → `multi-survey-ux`
- 나머지 → `other-feature`

**A안 (대안)**:
- Google Form에 "어느 기능?"을 Dropdown 또는 Radio button 추가
- 응답자가 제출 시 자동 선택 (구버전 응답은 수동 처리)
