# 위노트 경쟁 분석 — 노트 & 서식 관리 중심

> **작성일**: 2026-04-20
> **분석 대상**: 위노트(WeeNote) v2.8.0 — 한국 Wee클래스 전문상담교사용 앱
> **사용자 포커스**: ① 노션 스타일 **노트** 기능 / ② **서식 관리** 기능(쌤핀은 HWPX·PDF·Excel 모두 지원 목표)
> **제외**: 상담일지/활동일지는 참고 수준 (쌤핀은 담임교사 대상이라 성격 다름)

---

## 0. 핵심 결론 (TL;DR)

쌤핀이 위노트에서 **배워서 차별화할 두 가지**:

1. **노션 스타일 블록 에디터** — 현재 쌤핀 메모는 포스트잇뿐이라 장문 기록이 불가능.
   위노트는 "섹션 → 페이지 → 블록" 3단 계층에 슬래시 커맨드(`/`)로 블록 타입을 빠르게 전환.
   → 쌤핀은 **"쌤핀 노트"** 라는 별도 메뉴로 신설하고, 학생/학급/과목 컨텍스트와 연결해 차별화.

2. **서식 관리 — "HWPX + PDF + Excel 통합 서식 라이브러리"**
   위노트는 PDF만 지원하는 단순한 "PDF 드로워". 쌤핀은 한국 교사의 실제 워크플로우에 맞춰
   **3가지 포맷을 모두 다루고, 학생 데이터 자동 채움(mail-merge), 즐겨찾기, 인쇄 직행**까지
   제공하면 위노트 대비 뚜렷한 우위.

쌤핀의 선결 과제: 현재 **PdfExporter가 스텁**([`src/infrastructure/export/PdfExporter.ts`](../src/infrastructure/export/PdfExporter.ts))
— `electronAPI.printToPDF()` 호출 한 줄뿐. 서식 관리를 구현하려면 **PDF 실제 저장/렌더링 로직 완성**이 필요.

## 1. 위노트 "노트" 기능 분석 (관찰 기반)

### 1-1. UI 구조 — 4단 레이아웃

```
┌─────────┬──────────┬──────────┬──────────────────────────┐
│ 사이드바 │ 섹션 목록 │ 페이지   │  에디터 (블록 기반)      │
│         │ (+추가)  │ 목록     │  / 슬래시 메뉴로 블록 삽입│
└─────────┴──────────┴──────────┴──────────────────────────┘
```

- **계층 구조**: `섹션(Section) → 페이지(Page) → 블록(Block)[]` — OneNote + Notion 혼합
- **드래그 핸들**(⠿) 아이콘으로 섹션/페이지 재정렬 추정
- **"저장됨" 인디케이터** + 프린터 아이콘이 에디터 상단 우측에 상시 노출 → **자동 저장 + 1-클릭 인쇄**가 핵심 워크플로우

### 1-2. 블록 타입 (슬래시 `/` 메뉴에서 확인된 전체 목록)

| 카테고리 | 블록 | 단축키 |
|---|---|---|
| 제목 | H1 / H2 / H3 | `Ctrl+Alt+1~3` |
| 기본 | 인용, 접을 수 있는 목록, 번호 목록, 글머리 기호, 체크리스트, 본문, 코드 블록, 구분선 | `Ctrl+Shift+6~9`, `Ctrl+Alt+0/C` |
| 고급 | 표 | - |
| 미디어 | 이미지(확인), 파일/동영상은 스크롤 잘려 불확실 | - |

**강점**: Notion 사용자에게 친숙. 키보드 중심 입력 가능.
**약점**: 태그/즐겨찾기/검색 UI가 화면에 드러나지 않음, 플레인 텍스트 상담일지와 통합도 낮아 보임.

## 2. 위노트 "서식 관리" 기능 분석

### 2-1. 매우 단순한 구조
- 갤러리형 빈 상태: 점선 카드 `+ 서식 추가` / 부연설명 **"PDF 파일만 가능"**
- 헤더 문구: *"자주 사용하는 PDF 서식을 등록하고 바로 인쇄할 수 있습니다."*

### 2-2. 한계점 (쌤핀의 기회)
| 위노트 | 한계 |
|---|---|
| PDF만 지원 | 한국 교사는 **HWPX**(한글 공문서), **Excel**(성적/대장), **PDF**(가정통신문 배포) 모두 씀 |
| 단순 파일 저장소 | 학생 데이터 자동 채움(mail merge) 없음 |
| 카테고리 미확인 | 분류·검색 UX 불명확 |
| 학년도·학급 컨텍스트 없음 | "내 학급 29명에게 각각 가정통신문" 같은 일괄 출력 없음 |

## 3. 쌤핀 현황 (비교 지점)

### 3-1. 메모 vs 노트
| 항목 | 쌤핀(현재 Memo) | 위노트(Note) | 제안 |
|---|---|---|---|
| 스타일 | 포스트잇(자유 배치) | 문서형 블록 에디터 | **병존** — Memo는 유지, 신규 "쌤핀 노트" 추가 |
| 계층 | 없음 (평면) | 섹션→페이지 2단 | 노트북→섹션→페이지 3단 (학기/과목 대응) |
| 포맷 | 굵기/이탤릭/밑줄/링크/이미지 | 블록 기반(H1-H3/리스트/체크/표/코드) | 쌤핀 블록 ⊇ 위노트 블록 |
| 태그/검색 | ❌ 없음 | ❌ 미확인 | 🆕 쌤핀 차별화 포인트 |
| 학생 연결 | ❌ 없음 | ❌ 없음 | 🆕 쌤핀만의 강점 가능 |

### 3-2. 서식 관리 전제 — PDF Exporter 상태
[`src/infrastructure/export/PdfExporter.ts`](../src/infrastructure/export/PdfExporter.ts)는 현재 **8줄 스텁**:
```ts
if (window.electronAPI) return window.electronAPI.printToPDF();
window.print(); return null;
```
HWPX/Excel은 완성되어 있음([`HwpxExporter.ts`](../src/infrastructure/export/HwpxExporter.ts), [`ExcelExporter.ts`](../src/infrastructure/export/ExcelExporter.ts)).
**→ 서식 관리 기능 구현 전 PDF 렌더러를 먼저 채워야 함.**

## 4. 제안 기능 ① — "쌤핀 노트" (노션 스타일)

### 4-1. 포지셔닝
- **이름(잠정)**: `쌤핀 노트` 또는 `학습 노트`
- **사이드바 위치**: 메모 바로 아래 새 항목 추가
- **포스트잇 메모와의 구분**:
  - **Memo** = 휘발성 아이디어, 책상 위 포스트잇 (기존 그대로)
  - **Note** = 정리된 기록, 바인더/노트북 (신규)

### 4-2. 데이터 모델 (Clean Architecture 준수)
`src/domain/entities/`에 신규:
```ts
// Notebook.ts
export interface Notebook {
  id: string;
  title: string;        // 예: "2026학년도 1학기 수업 일지"
  icon?: string;
  context?: NoteContext; // 학급/과목/학생 연결 (차별화!)
  createdAt: string;
  updatedAt: string;
}

// NoteSection.ts
export interface NoteSection {
  id: string;
  notebookId: string;
  title: string;
  order: number;
}

// NotePage.ts
export interface NotePage {
  id: string;
  sectionId: string;
  title: string;
  blocks: NoteBlock[];
  tags: string[];       // 🆕 쌤핀 차별화 (위노트 없음)
  pinned: boolean;
  updatedAt: string;
}

// NoteBlock.ts
export type NoteBlockType =
  | 'heading1' | 'heading2' | 'heading3'
  | 'paragraph' | 'bullet' | 'numbered' | 'checklist' | 'toggle'
  | 'quote' | 'divider' | 'code' | 'table' | 'image'
  | 'callout'           // 🆕 강조 박스
  | 'student-ref'       // 🆕 학생 멘션 @김철수
  | 'record-embed';     // 🆕 담임메모 임베드
```

### 4-3. 위노트 대비 차별화 (★ 중요)
| 기능 | 위노트 | 쌤핀 제안 |
|---|---|---|
| 블록 타입 | 11종 | 위노트 전부 + **callout / 학생 멘션 / 담임메모 임베드** |
| 태그 | ❌ | ✅ 페이지당 태그 (예: `#학급경영 #진로`) |
| 검색 | ❌ | ✅ 전문(full-text) 검색 + 태그 필터 |
| 학생 연결 | ❌ | ✅ `@학생명` 멘션 → 학생 프로필로 점프 |
| 담임메모 연결 | ❌ | ✅ 페이지 내에서 `/기록삽입` → StudentRecord 블록 임베드 |
| 서식 연결 | ❌ | ✅ 노트에서 바로 "이 페이지를 HWPX/PDF/Excel 서식으로 저장" |
| 내보내기 | PDF 1클릭 | HWPX + PDF + Excel + Markdown 4종 |

### 4-4. UI 레이아웃 (쌤핀 디자인 톤 유지)
```
┌────────────┬──────────────┬──────────────┬─────────────────────┐
│ 사이드바    │ 노트북 목록  │ 섹션/페이지  │ 블록 에디터          │
│ (기존)     │ + 검색창     │ 트리         │ ┌─────────────────┐ │
│            │              │              │ │ 제목             │ │
│            │              │              │ ├─────────────────┤ │
│            │              │              │ │ 본문 블록들     │ │
│            │              │              │ │ / 로 블록 추가  │ │
│            │              │              │ └─────────────────┘ │
│            │              │              │ 저장됨 · HWPX/PDF   │

```
- 색상: 기존 `sp-bg / sp-surface / sp-card / sp-accent` 토큰 그대로
- 블록 호버 시 좌측에 `⠿` 드래그 핸들 + `+` 블록 추가 버튼 (Notion/위노트 공통 패턴)

### 4-5. 기술 스택 제안
- 에디터 코어: **TipTap** (ProseMirror 기반, React 친화, 블록/슬래시 메뉴 확장 풍부)
  - 대안: BlockNote (Notion 클론 라이브러리, 더 가볍지만 커스터마이징 제약)
- 저장: 기존 `IStoragePort` 그대로 → `notebooks.json`, `notes/{pageId}.json` 분리 저장 권장 (페이지당 1파일)
- 검색: `lunr.js` 또는 `flexsearch` (로컬 전문검색, 오프라인 유지)

### 4-6. 구현 단계 (점진적)
- **Phase 1 (MVP, 2주)**: 노트북/섹션/페이지 CRUD + H1-H3/본문/리스트/체크/구분선 블록 + 자동 저장
- **Phase 2 (1주)**: 표/코드/인용/토글/이미지 블록 + 슬래시 커맨드 + 키보드 단축키
- **Phase 3 (1주)**: 태그 + 검색 + 고정(pinned) + Markdown 내보내기
- **Phase 4 (차별화, 2주)**: 학생 멘션 / 담임메모 임베드 / Callout / 서식 저장 연동

## 5. 제안 기능 ② — "서식 관리" (HWPX + PDF + Excel 통합)

### 5-1. 컨셉
한국 교사의 **실제** 서식 사용 패턴:
- **HWPX**: 공문, 가정통신문, 생활기록부 초안 → 한글 폼 그대로 쓰고 인쇄
- **Excel**: 성적 산출, 상담관리대장, 출석부 → 계산·필터·반별 탭 필요
- **PDF**: 완성된 문서 배포, 학부모 안내문 → 수정 불가 형태로 나눠주기

위노트의 **"PDF 드로워"** 수준을 넘어, 쌤핀은 **"교사의 서식 라이브러리 + 자동 채움 + 일괄 출력"** 으로.

### 5-2. 데이터 모델
```ts
// FormTemplate.ts (Template 이름 기존 RecordTemplate과 구분 위해)
export type FormFormat = 'hwpx' | 'pdf' | 'excel';
export type FormCategory = '가정통신문' | '공문서' | '성적/평가' | '생활기록' | '상담' | '기타';

export interface FormTemplate {
  id: string;
  name: string;                      // "2학기 학부모 상담 안내문"
  format: FormFormat;                // hwpx/pdf/excel
  category: FormCategory;
  filePath: string;                  // userData/forms/{id}.{ext}
  thumbnail?: string;                // 첫 페이지 썸네일 (PNG base64)
  starred: boolean;                  // 즐겨찾기
  mergeFields?: MergeField[];        // 🆕 자동 채움용 (아래)
  tags: string[];
  createdAt: string;
  lastUsedAt?: string;               // "최근 사용" 정렬용
  useCount: number;
}

// MergeField — mail-merge용
export interface MergeField {
  placeholder: string;  // "{{학생이름}}", "{{학급}}" 등
  source: 'student.name' | 'student.number' | 'class.name'
        | 'class.teacher' | 'date.today' | 'custom';
  customValue?: string;
}
```

### 5-3. 주요 기능 (위노트 대비)
| 기능 | 위노트 | 쌤핀 제안 |
|---|---|---|
| 지원 포맷 | PDF만 | **HWPX + PDF + Excel** |
| 등록 방식 | 파일 업로드 | 파일 업로드 + **쌤핀 노트 → 서식으로 저장** |
| 카테고리 | 불분명 | 6개 기본 카테고리 + 사용자 정의 |
| 즐겨찾기 | ❌ | ⭐ 자주 쓰는 서식 고정 |
| 미리보기 | 불분명 | 첫 페이지 썸네일 자동 생성 |
| 학생 데이터 채움 | ❌ | ✅ **`{{학생이름}}` 등 플레이스홀더 → 학급 전체 일괄 치환** |
| 일괄 인쇄 | ❌ | ✅ "학급 29명 가정통신문 1번에 출력" |
| 최근 사용 | ❌ | ✅ `lastUsedAt` 정렬 |
| 기본 제공 서식 | 없음 | 10종 내장 (아래) |

### 5-4. 쌤핀 내장 기본 서식 10종 (제안)
차별화 강화를 위해 **한국 교사가 자주 쓰는 서식**을 내장:

| # | 서식명 | 포맷 | 카테고리 |
|---|---|---|---|
| 1 | 학부모 상담 신청 안내문 | HWPX | 가정통신문 |
| 2 | 현장체험학습 안내 및 신청서 | HWPX | 가정통신문 |
| 3 | 학급 시간표 | Excel | 생활기록 |
| 4 | 출석부 (월간) | Excel | 생활기록 |
| 5 | 자리 배치표 | PDF | 생활기록 |
| 6 | 학생 인적사항 카드 | HWPX | 생활기록 |
| 7 | 담임메모 요약 보고서 | PDF | 상담 |
| 8 | 학급 좌석 뽑기 결과 | PDF | 기타 |
| 9 | 수행평가 채점표 | Excel | 성적/평가 |
| 10 | 학급 임원 선거 개표표 | Excel | 기타 |

이미 HWPX/Excel exporter가 있으니 **기존 내보내기 로직을 서식 템플릿으로 재활용** 가능 — 개발 부담 적음.

### 5-6. 핵심 워크플로우 — "일괄 채움 인쇄"
1. 교사가 `학부모 상담 신청 안내문.hwpx` 선택
2. "학급 전체에 일괄 채움" 클릭
3. 쌤핀이 학급 학생 29명을 순회하며 `{{학생이름}}`, `{{번호}}`, `{{학급}}` 자동 치환
4. **29개 HWPX 파일** 또는 **1개 통합 PDF (29페이지)** 생성
5. 바로 인쇄 or 폴더에 저장

→ 위노트는 PDF 1장 뽑는 수준. 쌤핀은 **"학급 29명치 한 번에"** 가능.

### 5-7. 기술 구현 요점
| 컴포넌트 | 라이브러리/방식 |
|---|---|
| HWPX 템플릿 치환 | 기존 `@ubermensch1218/hwpxcore` 활용 + `{{placeholder}}` 치환 로직 |
| Excel 템플릿 치환 | 기존 `exceljs`의 `getCell().value` 순회 치환 |
| PDF 템플릿 치환 | `pdf-lib` (폼 필드 채움) + `pdfjs-dist` (썸네일 생성) |
| 썸네일 생성 | HWPX → HWPX→PDF 변환 후 첫 페이지, Excel → 첫 시트 canvas, PDF → 첫 페이지 |
| 파일 저장 | `app.getPath('userData')/forms/{id}.{ext}` (기존 IStoragePort 확장) |

**선결 조건**: 현재 8줄 스텁인 [`PdfExporter.ts`](../src/infrastructure/export/PdfExporter.ts)를 **`pdf-lib` 기반 전면 재작성**.

### 5-8. 구현 단계
- **Phase 1 (MVP, 1.5주)**: 서식 등록/목록/삭제/즐겨찾기 + 포맷 필터 + 썸네일
- **Phase 2 (2주)**: 기본 내장 서식 10종 + `pdf-lib` 기반 PDF 렌더/채움
- **Phase 3 (2주)**: 학생 데이터 mail-merge + 일괄 채움 인쇄
- **Phase 4 (1주)**: 쌤핀 노트 → "서식으로 저장" 연동

## 8. 리스크 & 판단 필요 지점
