### 2.2 Warning (의미 있는 불일치) — **3건**

#### W1. `SpreadsheetView` props에 설계에 없던 `inlineFooter` 슬롯 추가
- **위치:** `src/adapters/components/Tools/Results/SpreadsheetView.tsx:23–25, 168–173`
- **설계:** §3.1 Props에는 `source`·`onClose`만 정의됨. inline 모드 하단 액션 영역 규격 없음.
- **구현:** `inlineFooter?: React.ReactNode` 슬롯으로 `<ResultSaveButton>` + "🆕 새 설문" 버튼 투입. `ToolMultiSurvey.tsx:1425–1439`에서 실제 활용 중.
- **영향:** 낮음. 기존 `ToolMultiSurvey.results` 단계의 "결과 저장 + 새 설문 시작" 액션을 유지하면서 SpreadsheetView를 재사용하기 위한 합리적 구조.
- **권장:** Design v0.3에서 §3.1 Props에 `inlineFooter` 추가, §5.1 ASCII 레이아웃에 하단 액션 바 표현 포함.

#### W2. `filterOption` / `searchKeyword` 탭 간 보존 정책 — 구현 패턴 차이
- **위치:** `src/adapters/components/Tools/Results/TableTab.tsx:24–27`
- **설계:** §3.1 State 표에 "`table → 타탭 이탈 시 리셋`"으로 명시. 부모가 state 보유 + 명시적 리셋.
- **구현:** `filterOption`·`searchKeyword`·`sortColumn`·`sortDir`를 `TableTab` 내부 `useState`로 보유. 탭 전환 시 unmount로 **결과적으로 리셋 동작은 일치**.
- **영향:** 낮음. 관측 가능한 동작은 동일. 단, 탭을 `display:none` 토글로 리팩터링할 경우 동작 변화 가능.
- **권장:** Design §3.1 State 표 각주에 "Table 상태는 TableTab 내부 state로 구현 — 탭 전환 시 unmount로 자연 리셋" 반영.

#### W3. `exportToolResultToExcel` switch에 discussion 타입 명시적 throw 추가
- **위치:** `src/infrastructure/export/ExcelExporter.ts:2126–2128`
- **설계:** §4.1은 `poll | survey | multi-survey | wordcloud`만 `switch` 처리, `default: throw` 패턴. discussion 타입 언급 없음.
- **구현:** `case 'valueline-discussion': case 'trafficlight-discussion': throw new Error(...)`.
- **영향:** 긍정적. `ToolResultData` union에 이미 discussion 타입 2종이 존재하므로 exhaustive 검사 관점에서 더 안전.
- **권장:** Design §4.1에 두 discussion 타입 `case` + 에러 메시지 명시.

### 2.3 Info (설계에 없었던 개선/명시화 — 문서 반영 권장) — **7건**

#### I1. ⭐ Portal(createPortal) 사용 — modal 모드 containing block 버그 방지
- **위치:** `src/adapters/components/Tools/Results/SpreadsheetView.tsx:177–192`
- **설계:** §3.1 "모달 모드" 설명에 `fixed inset-0 bg-black/60` 오버레이만 명시. Portal 언급 없음.
- **구현:** `createPortal(..., document.body)`로 document.body에 렌더.
- **영향:** **필수**. ToolLayout 계열 상위 컨테이너에 `transform`이 걸려 있으면 CSS 사양상 `position: fixed`의 containing block이 해당 조상으로 바뀜. 이 수정이 없으면 모달이 시각적으로 잘리거나 오프셋됨. Step 5 브라우저 검증에서 실제로 발견·수정.
- **권장:** Design §3.1에 "Modal 모드: `createPortal(document.body)` 필수 — 상위 컨테이너 `transform` 시 containing block 트랩 방지" 명시.

#### I2. 라이트 테마 호환 보정 — `bg-sp-accent/70` → `bg-sp-accent`
- **위치:** Results/ 4개 파일 모두 투명도 없는 accent만 사용 (grep 결과 `bg-sp-accent/` 0건)
- **설계:** §3.1 "디자인 토큰 사용" 항목에 `sp-accent`(활성 탭·주 CTA)만 명시.
- **구현:** 불투명 accent 사용 → 라이트/다크 양쪽 호환.
- **배경:** 프로젝트 기본 테마가 Notion 스타일 light (sp-bg=#ffffff)였으며, 투명도 있는 accent(bg-sp-accent/70)는 light 배경 위에서 대비 부족.
- **권장:** Design §3.1에 "`sp-accent` 투명도 변형(`/20`, `/70`)은 라이트 테마에서 대비 저하 → 불투명 accent + hover 변형만 사용" 가이드 추가.

#### I3. Excel 반환 타입 — `ArrayBuffer` 캐스트로 통일
- **위치:** `src/infrastructure/export/ExcelExporter.ts:2094–2131`
- **설계:** §4.1 주석은 "반환 타입은 ExcelJS 관례에 위임"으로 Buffer/ArrayBuffer 모두 허용 톤.
- **구현:** `(await workbook.xlsx.writeBuffer()) as ArrayBuffer` 캐스트로 기존 exporter 16개 함수와 통일.
- **권장:** Design §4.1 반환 타입을 `Promise<ArrayBuffer>`로 고정 표기.

#### I4. `SpreadsheetView` 'multi-survey' 타입 가드 추가
- **위치:** `src/adapters/components/Tools/Results/SpreadsheetView.tsx:39–45, 79–85`
- **설계:** §3.1 내부 정규화 예시는 단순 캐스트.
- **구현:** `if (source.result.data.type === 'multi-survey') return source.result.data; return null;` + null 체크 후 "멀티 설문 결과가 아닙니다" fallback.
- **권장:** Design §3.1 내부 정규화 코드 블록을 타입 가드 형태로 교체.

#### I5. Sheet 2 응답자/제출시각 2열 분리
- **위치:** `ExcelExporter.ts:1954–1955`, `IndividualTab.tsx:60–64`
- **설계:** §4.2 Sheet 2 "A열: 응답자 라벨 + 제출 시각"에서 1열로 기술.
- **구현:** `응답자`(A열)와 `제출 시각`(B열)을 분리. 시간 형식 `YYYY-MM-DD HH:mm`.
- **영향:** 긍정적 — Excel 정렬/필터 용이.
- **권장:** Design §4.2 표를 "A열: 응답자 / B열: 제출 시각 / C~ : Q1…" 2열 분리로 명시.

#### I6. `totalParticipants` / `completionRate` 유틸 함수를 추가 export
- **위치:** `src/domain/rules/toolResultAggregation.ts:201–226`
- **설계:** §2.2는 `aggregateQuestion`, `aggregateAll`만 시그니처 제시. 완료율/참여 수는 §5.1 UI에만 언급.
- **구현:** 두 순수 함수를 추가 export. SummaryTab + ExcelExporter에서 공유 사용.
- **권장:** Design §2.2에 두 함수 시그니처 추가 (multi-choice required 빈 배열을 미완료로 처리하는 엣지 케이스 포함).

#### I7. `exportToolResultToExcel` 호출 경로 분리
- **위치:** `SpreadsheetView.tsx:55–62`
- **설계:** §4.4는 `data.title` 기반 파일명만 한 가지 경로 예시.
- **구현:** `source.mode === 'inline' ? exportMultiSurveyDataToExcel(data, ...) : exportToolResultToExcel(source.result, ...)` 명확 분기.
- **권장:** Design §4.4 코드 예시를 두 경로 switch로 확장.
