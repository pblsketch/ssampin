## 1. 개요

| 항목 | 내용 |
|------|------|
| **기능명** | 나이스 시간표 자동 불러오기 |
| **목적** | 수동 입력 부담 제거, 학기 초 시간표 설정 간소화 |
| **대상 사용자** | 초등학교·중학교·고등학교 교사 |
| **핵심 가치** | 나이스에 이미 등록된 시간표를 클릭 한 번으로 쌤핀에 동기화 |

### 1.1 배경

쌤핀의 시간표는 현재 `TimetableEditor`를 통한 **수동 입력**만 지원한다. 매 학기 초마다 교사가 일일이 교과명·반을 입력해야 하는데, 나이스 교육정보 개방 포털(open.neis.go.kr)의 공개 API를 활용하면 이 과정을 자동화할 수 있다.

### 1.2 현재 코드 구조

```
src/
├── stores/
│   └── useScheduleStore.ts    # teacherSchedule, classSchedule (Zustand)
├── components/
│   ├── Timetable/
│   │   ├── Timetable.tsx       # mode: 'class' | 'teacher', 월~금 표시
│   │   └── TimetableEditor.tsx # 수동 편집 UI
│   └── Settings/
│       └── Settings.tsx        # 현재 나이스 관련 설정 없음
└── utils/
    └── storage.ts              # readData(), writeData()
```

- **프레임워크**: React + TypeScript + Zustand + Tailwind CSS + Electron
- **데이터 타입**: `TeacherScheduleData`, `ClassScheduleData`
- **저장 방식**: `readData`/`writeData`로 로컬 JSON 파일

## 2. 사용자 스토리

| # | 스토리 | 우선순위 |
|---|--------|----------|
| US-01 | 교사는 설정 화면에서 **학교명을 검색**하고 내 학교를 선택할 수 있다. | P0 |
| US-02 | 교사는 **학년과 반**을 선택할 수 있다. | P0 |
| US-03 | "시간표 불러오기" 버튼을 누르면 **나이스에서 주간 시간표가 자동으로 채워진다**. | P0 |
| US-04 | 불러온 시간표를 **수동으로 미세 조정**할 수 있다. | P0 |
| US-05 | **오프라인 환경**에서도 마지막으로 불러온 시간표를 볼 수 있다. | P0 |
| US-06 | 새 학기가 시작되면 **시간표를 다시 불러올** 수 있다. | P1 |
| US-07 | 교사 모드에서 **여러 반의 시간표를 조합**하여 내 수업만 볼 수 있다. | P1 |
| US-08 | API 키 없이도 **제한적으로 사용**할 수 있다 (1일 5회). | P2 |

### 2.1 사용자 시나리오 (Happy Path)

```
1. 교사가 시간표 편집 화면에서 "나이스에서 불러오기" 클릭
2. 모달 열림 → 학교명 입력 → 검색 결과에서 학교 선택
3. 학년 선택 → 반 목록 자동 로드 → 반 선택
4. "불러오기" 클릭 → 로딩 → 시간표 자동 채움
5. 필요시 수동으로 수정 → 저장
```

## 3. 기능 요구사항

### FR-01: 나이스 API 키 설정

| 항목 | 내용 |
|------|------|
| **위치** | Settings.tsx > "나이스 연동" 섹션 |
| **동작** | API 키 입력 필드 (선택사항) |
| **저장** | 로컬 설정 파일에 암호화 없이 저장 (Electron 로컬 전용) |
| **참고** | API 키 없이도 1일 5회 호출 가능 (KEY 파라미터 생략 시) |

### FR-02: 학교 검색 및 선택

| 항목 | 내용 |
|------|------|
| **API** | `GET https://open.neis.go.kr/hub/schoolInfo?SCHUL_NM={검색어}&Type=json&KEY={apiKey}` |
| **반환 데이터** | `ATPT_OFCDC_SC_CODE`(교육청코드), `SD_SCHUL_CODE`(학교코드), `SCHUL_KND_SC_NM`(학교급) |
| **UI** | 검색 입력 → 결과 목록(학교명, 주소, 학교급) → 클릭하여 선택 |
| **저장** | 선택한 학교 정보를 설정에 저장 (다음 접속 시 재사용) |
| **디바운스** | 입력 후 300ms 디바운스 적용 |

### FR-03: 학년/반 선택

| 항목 | 내용 |
|------|------|
| **API** | `GET https://open.neis.go.kr/hub/classInfo?KEY={apiKey}&ATPT_OFCDC_SC_CODE={코드}&SD_SCHUL_CODE={코드}&AY={학년도}&GRADE={학년}` |
| **반환 데이터** | `CLASS_NM`(반 이름) 목록 |
| **UI** | 학년 드롭다운 → 반 드롭다운 (classInfo API로 자동 로드) |
| **학년 범위** | 초등: 1~6, 중등: 1~3, 고등: 1~3 (학교급에 따라 자동 설정) |

### FR-04: 학급 시간표 불러오기 (Class Mode)

| 항목 | 내용 |
|------|------|
| **API (초등)** | `GET https://open.neis.go.kr/hub/elsTimetable` |
| **API (중등)** | `GET https://open.neis.go.kr/hub/misTimetable` |
| **API (고등)** | `GET https://open.neis.go.kr/hub/hisTimetable` |
| **파라미터** | `KEY`, `ATPT_OFCDC_SC_CODE`, `SD_SCHUL_CODE`, `AY`, `SEM`, `GRADE`, `CLASS_NM`, `TI_FROM_YMD`, `TI_TO_YMD` |
| **반환 데이터** | `PERIO`(교시), `ITRT_CNTNT`(과목명), `ALL_TI_YMD`(날짜, YYYYMMDD) |
| **변환** | 날짜 → 요일 매핑, 교시 → 행 인덱스, 과목명 → `ClassScheduleData` 형식 |
| **기간** | 최근 1주일 또는 사용자가 지정한 주 |

### FR-05: 교사 시간표 불러오기 (Teacher Mode)

| 항목 | 내용 |
|------|------|
| **방법 A** | 교사가 담당 학년/반 복수 선택 → 해당 학급 시간표 전부 조회 → 교사명으로 필터링 |
| **방법 B** | 학급 시간표 불러온 뒤, 수동으로 교과명+반 매핑 |
| **한계** | 나이스 API는 교사 시간표를 직접 제공하지 않음 |
| **UI** | 복수 학년/반 체크박스 선택 → 불러오기 → 수동 보완 안내 |

### FR-06: 데이터 변환 및 저장

| 항목 | 내용 |
|------|------|
| **입력** | 나이스 API JSON 응답 |
| **출력** | `TeacherScheduleData` 또는 `ClassScheduleData` |
| **저장** | `writeData()`를 통해 로컬 JSON 파일에 저장 |
| **요일 매핑** | `ALL_TI_YMD` → `Date.getDay()` → 월(0)~금(4) |
| **교시 매핑** | `PERIO` → 배열 인덱스 (1교시=index 0) |

### FR-07: 불러오기 후 수동 편집

| 항목 | 내용 |
|------|------|
| **동작** | 불러오기 완료 후 기존 `TimetableEditor` UI 그대로 사용 |
| **수정** | 과목명, 반 등 자유롭게 편집 가능 |
| **병합** | 기존 데이터가 있을 경우 "덮어쓰기/병합" 선택 다이얼로그 |

### FR-08: 학기/학년 변경 시 다시 불러오기

| 항목 | 내용 |
|------|------|
| **트리거** | 설정에서 학년도/학기 변경 시, 또는 수동 "다시 불러오기" 버튼 |
| **동작** | 기존 시간표 백업 후 새로 불러오기 |
| **학기 자동 감지** | 현재 날짜 기준: 3~7월 = 1학기, 8~2월 = 2학기 |

### FR-09: 에러 처리

| 에러 유형 | 처리 방법 |
|-----------|-----------|
| 네트워크 오류 | "인터넷 연결을 확인해주세요" 토스트 메시지 |
| API 키 무효 | "API 키가 올바르지 않습니다. 설정에서 확인해주세요" |
| 데이터 없음 | "해당 기간의 시간표 데이터가 없습니다. 학기 중인지 확인해주세요" |
| API 호출 제한 | "일일 호출 횟수를 초과했습니다. API 키를 등록하면 더 많이 사용할 수 있습니다" |
| 학교 검색 결과 없음 | "검색 결과가 없습니다. 학교명을 다시 확인해주세요" |

### 5.1 데이터 변환 상세

**나이스 API 응답 (raw)**:
```json
{
  "PERIO": "1",
  "ITRT_CNTNT": "국어",
  "ALL_TI_YMD": "20260302"
}
```

**변환 후 (ClassScheduleData 형식)**:
```typescript
// classSchedule[dayIndex][periodIndex] = "국어"
// dayIndex: 0(월)~4(금), periodIndex: 0(1교시)~
```

**변환 후 (TeacherScheduleData 형식)**:
```typescript
// teacherSchedule[dayIndex][periodIndex] = { subject: "국어", class: "2-3" }
```

## 6. 나이스 API 상세

### 6.1 학교기본정보 API

```
GET https://open.neis.go.kr/hub/schoolInfo
  ?SCHUL_NM={검색어}
  &Type=json
  &KEY={apiKey}        // 선택
  &pIndex=1
  &pSize=20
```

### 6.2 학급정보 API

```
GET https://open.neis.go.kr/hub/classInfo
  ?KEY={apiKey}
  &Type=json
  &ATPT_OFCDC_SC_CODE={교육청코드}
  &SD_SCHUL_CODE={학교코드}
  &AY={학년도}         // e.g., "2026"
  &GRADE={학년}        // e.g., "2"
  &pIndex=1
  &pSize=100
```

### 6.3 시간표 API

```
GET https://open.neis.go.kr/hub/{endpoint}Timetable
  ?KEY={apiKey}
  &Type=json
  &ATPT_OFCDC_SC_CODE={교육청코드}
  &SD_SCHUL_CODE={학교코드}
  &AY={학년도}
  &SEM={학기}          // "1" 또는 "2"
  &GRADE={학년}
  &CLASS_NM={반}
  &TI_FROM_YMD={시작일}  // YYYYMMDD
  &TI_TO_YMD={종료일}    // YYYYMMDD
  &pIndex=1
  &pSize=1000
```

| 학교급 | endpoint |
|--------|----------|
| 초등학교 | `els` |
| 중학교 | `mis` |
| 고등학교 | `his` |

## 8. 제약사항 및 리스크

## 10. 향후 확장 가능성

- **자동 동기화**: 주 1회 자동으로 시간표 변경 감지 및 알림
- **교사 시간표 자동 매칭**: 교사명으로 나이스 데이터 필터링 (API에서 교사 정보 제공 시)
- **시간표 공유**: 같은 학교 교사 간 시간표 공유
- **시간표 변경 이력**: 나이스 데이터와 현재 데이터 비교 → 변경점 하이라이트
