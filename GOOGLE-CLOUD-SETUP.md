# Google Cloud 설정 가이드 — 쌤핀(SsamPin) 구글 캘린더 연동

> 구글 캘린더 연동을 위해 Google Cloud Console에서 OAuth 2.0 인증 정보를 발급받는 절차입니다.

---

## Step 1: 프로젝트 생성

1. [https://console.cloud.google.com](https://console.cloud.google.com) 접속
2. 상단 프로젝트 선택 드롭다운 → **새 프로젝트** 클릭
3. 프로젝트 이름: `쌤핀 (SsamPin)` 입력
4. **만들기** 클릭 후 해당 프로젝트로 전환

---

## Step 2: Google Calendar API 활성화

1. 왼쪽 메뉴 → **API 및 서비스** → **라이브러리**
2. 검색창에 `Google Calendar API` 입력
3. 검색 결과에서 **Google Calendar API** 클릭
4. **사용 설정** 버튼 클릭

---

## Step 3: OAuth 동의 화면 설정

1. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. 사용자 유형: **외부** 선택 → **만들기**
3. 앱 정보 입력:
   - 앱 이름: `쌤핀`
   - 사용자 지원 이메일: 본인 구글 계정
   - 개발자 연락처 이메일: 본인 이메일
4. **저장 후 계속** 클릭
5. 범위(Scopes) 추가:
   - **범위 추가 또는 삭제** 클릭
   - `https://www.googleapis.com/auth/calendar` 검색 후 선택
   - **업데이트** 클릭
6. **저장 후 계속** 클릭
7. 테스트 사용자 추가:
   - **사용자 추가** 클릭
   - 사용할 구글 계정 이메일 입력 (최대 100명)
8. **저장 후 계속** → 요약 확인 후 완료

---

## Step 4: OAuth 2.0 클라이언트 ID 생성

1. 왼쪽 메뉴 → **API 및 서비스** → **사용자 인증 정보**
2. **사용자 인증 정보 만들기** → **OAuth 클라이언트 ID** 클릭
3. 애플리케이션 유형: **데스크톱 애플리케이션** 선택
4. 이름: `쌤핀 데스크톱` 입력
5. **만들기** 클릭
6. 팝업에서 **클라이언트 ID**와 **클라이언트 보안 비밀** 복사
7. 프로젝트 루트의 `.env` 파일에 입력:

```
VITE_GOOGLE_CLIENT_ID=복사한-클라이언트-ID.apps.googleusercontent.com
VITE_GOOGLE_CLIENT_SECRET=복사한-클라이언트-보안-비밀
```

> `.env.example` 파일을 복사하여 `.env`로 저장한 후 값을 채우세요.
> `.env` 파일은 절대 Git에 커밋하지 마세요 (`.gitignore`에 등록되어 있습니다).

---

## Step 5: 테스트 모드 확인 (개발 단계)

- OAuth 동의 화면에서 **게시 상태: 테스트**로 시작합니다.
- Step 3에서 등록한 테스트 사용자 계정만 앱을 사용할 수 있습니다.
- 최대 100명의 테스트 사용자를 등록할 수 있습니다.

> ⚠️ 이 상태는 개발용입니다. 배포 전 반드시 프로덕션 전환이 필요합니다 (Step 6 참고).

---

## Step 6: 프로덕션 전환 준비 (개발과 병행)

> ⚠️ 쌤핀은 180명+ 사용자를 목표로 합니다. 테스트 모드의 100명 한계를 초과할 수 있습니다.
> 개발 초기에는 테스트 모드로 진행하고, **Phase 1 완료 시점**에 프로덕션 전환 신청을 진행하세요.

### 사전 준비물

| 항목 | 설명 |
|------|------|
| 개인정보처리방침 URL | 랜딩페이지에 `/privacy` 페이지 추가 |
| 앱 홈페이지 URL | 기존 랜딩페이지 URL |
| 앱 로고 | 120×120 픽셀 이상 PNG |
| 데모 영상 | 1~2분 분량, YouTube에 업로드 |
| scope 사용 목적 설명문 | 영문으로 작성 (캘린더 데이터를 어떻게 사용하는지) |

### 전환 절차

1. OAuth 동의 화면 → **앱 게시** 클릭
2. Google의 검토 요청 양식 작성
3. 검토 통과 후 모든 구글 계정 사용자가 앱을 사용할 수 있게 됨

### 비용

- **전부 무료** — `calendar` scope는 Sensitive scope이지만 유료 보안 평가(CASA) 대상이 아닙니다.
